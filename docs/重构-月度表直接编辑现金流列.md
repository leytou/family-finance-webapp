# 重构：月度表直接编辑现金流列

## 背景

当前现金流项目通过左侧面板管理（添加项目→展开→添加金额段→拖拽时间范围），操作繁琐。重构为直接在月度表上插入/编辑列，所见即所得，大幅简化交互。

同时移除 `currentSavings`（用累计列第一个月锚点替代），移除左侧面板。

## 新交互模型

- 月度表表头有「+」按钮，点击插入新列（位于累计列之前）
- 列头可点击编辑名称，hover 显示删除按钮
- 单元格直接输入数字：正数=收入（绿色），负数=支出（红色）
- **延续规则**：某月填入值后自动延续到后续月份，直到遇到 0（停止）或新非零值（切换）
- 累计列 = 上月累计 + 本月所有自定义列之和 + 投资收益
- 累计列可编辑（锚点功能保留）
- 系统参数（年化收益率、起始月份）移入顶部 header

## 新数据模型

```typescript
// 替代 CashFlowItem + AmountSegment
interface FlowColumn {
  id: string
  name: string
  entries: Record<number, number>  // 稀疏存储，key=YYYYMM，只存用户手动编辑的值
}

// SystemParams 移除 currentSavings
interface SystemParams {
  startMonth: number
  annualRate: number
}

// PlanData: items → columns, version 1 → 2
interface PlanData {
  version: number
  systemParams: SystemParams
  columns: FlowColumn[]
  anchors: MonthlyAnchor[]
}

// MonthResult 移除 incomeItems/expenseItems，改为通用 columnValues
interface MonthResult {
  month: number
  columnValues: { id: string; name: string; amount: number; isEdited: boolean }[]
  totalFlow: number
  investReturn: number
  netSavings: number
  cumSavings: number
  isAnchor: boolean
}
```

## 延续规则算法

给定 FlowColumn 和某月，求该月显示值：

1. 若 `entries[month]` 存在 → 返回该值，标记 `isEdited=true`
2. 否则向前查找最近的 entry（YYYYMM < 当前月）：
   - 找到且值非零 → 延续该值，标记 `isEdited=false`
   - 找到且值为 0 → 显示 0，`isEdited=false`
   - 未找到任何 entry → 显示 0，`isEdited=false`

## 实现步骤

### Step 1: 类型更新

**文件:** `src/types.ts`

- 新增 `FlowColumn` 接口
- `SystemParams` 移除 `currentSavings`
- `PlanData`: `items: CashFlowItem[]` → `columns: FlowColumn[]`
- `MonthResult`: 移除 `incomeItems`/`expenseItems`/`totalIncome`/`totalExpense`，新增 `columnValues` 和 `totalFlow`
- 保留 `MonthlyAnchor` 不变

### Step 2: 计算引擎重写

**文件:** `src/composables/useCalculation.ts`

- 新增 `resolveColumnValue(column: FlowColumn, month: number, allMonthsSoFar: number[])` 实现延续逻辑
- 重写 `calculate(plan: PlanData): MonthResult[]`：
  - 首月 prevCum = 0（无 currentSavings，靠锚点设置初始值）
  - `columnValues = columns.map(col => resolveColumnValue(col, month))`
  - `totalFlow = sum(columnValues.amount)`
  - `investReturn = prevCum * annualRate / 12`
  - `netSavings = totalFlow + investReturn`
  - `cumSavings = anchor ? anchor.actualSavings : prevCum + netSavings`
- 删除旧函数：`getMonthlyItems`、`getActiveAmount`、`aggregateByName`

### Step 3: Store 重构

**文件:** `src/composables/useStore.ts`

- `createDefault()` 更新：`version: 2`，无 `currentSavings`，`columns: []`
- 不兼容旧数据，加载旧格式时直接返回 `createDefault()`（reset）
- 新增操作函数：
  - `addColumn(name?: string)` — 生成 ID，默认名 "新列"
  - `removeColumn(id: string)` — 按 ID 过滤
  - `renameColumn(id: string, name: string)` — 修改名称
  - `updateColumnEntry(colId: string, month: number, value: number | null)` — 设置或删除 entry
- 删除：`addItem`、`removeItem`
- 更新 `isValidPlanData()` 仅验证 v2 格式，旧格式视为无效数据直接重置

### Step 4: MonthlyTable 重写

**文件:** `src/components/MonthlyTable.vue`

表头结构：
```
| 月份 | [col1.name] | [col2.name] | ... | [+] | 理财 | 净储蓄 | 累计 |
```

交互：
- 列头双击进入重命名模式（inline input，Enter 确认，Escape 取消）
- 列头 hover 显示删除按钮（点击后 confirm 再删除）
- 「+」按钮调用 `addColumn()`
- 单元格点击进入编辑（类似现有累计列编辑）
  - 编辑状态：`editingCell: { columnId: string, month: number } | null`
  - 确认 → `updateColumnEntry(colId, month, num)`
  - 清空输入 → `updateColumnEntry(colId, month, null)` 移除 entry，恢复延续值
- 已编辑单元格（isEdited=true）浅蓝底色区分
- 正值绿色，负值红色，0 不着色
- 保留累计列编辑功能（锚点）

组件内部直接调用 `useStore()` 获取 columns 和操作函数。

### Step 5: AnnualTable 适配

**文件:** `src/components/AnnualTable.vue`

- 移除 `incomeItems` / `expenseItems` 分类聚合

- MonthlyTable 不再通过 emit 转发锚点事件，内部直接调用 store
