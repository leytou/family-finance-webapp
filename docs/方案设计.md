# 家庭财务规划网页应用 — 技术设计方案

## Context

现有项目是一个 Python 脚本生成 Excel 的财务规划工具，需求文档要求重写为浏览器本地运行的网页应用。新版本采用通用化的"现金流项目"模型，用户可自由添加任意收入/支出项目，取代原来硬编码的场景。新增月度锚点校正和网格滑动选时间段交互。

---

## 1. 技术栈

| 层面 | 选择 | 理由 |
|------|------|------|
| 框架 | Vue 3 + Composition API | 轻量、响应式系统天然适合实时联动计算 |
| 语言 | TypeScript | 数据模型复杂，类型安全有价值 |
| 构建 | Vite | 开发体验好，零配置即可启动 |
| 样式 | UnoCSS (Tailwind 兼容) | 紧凑表格需要精细控制间距，原子 CSS 高效 |
| 状态 | Vue reactive + watchEffect | 60 个月数据量小，不需要额外状态管理库 |
| 持久化 | localStorage + JSON | 需求明确要求 |
| 测试 | Vitest + @vue/test-utils | Vite 原生测试框架，快速、零配置 |

不引入 UI 组件库 — 表格和面板都是自定义的紧凑布局，组件库反而增加约束。

开发方法：**TDD（测试驱动开发）**— 每个模块先写测试，再实现代码，确保逻辑正确。

## 2. 数据模型

```typescript
// 金额段：一个项目在某个时间范围内的金额
interface AmountSegment {
  amount: number
  startMonth: number  // YYYYMM 格式，如 202602
  endMonth: number    // YYYYMM 格式，如 203012
}

// 现金流项目
interface CashFlowItem {
  id: string
  name: string
  type: 'income' | 'expense'
  segments: AmountSegment[]
}

// 月度锚点
interface MonthlyAnchor {
  month: number       // YYYYMM
  actualSavings: number
}

// 系统参数
interface SystemParams {
  currentSavings: number
  startMonth: number       // YYYYMM，规划起始月
  annualRate: number       // 年化理财收益率，默认 0.025
}

// 完整规划数据（localStorage 存储单元）
interface PlanData {
  version: number
  systemParams: SystemParams
  items: CashFlowItem[]
  anchors: MonthlyAnchor[]
}
```

## 4. 核心模块设计

### 4.1 计算引擎 (useCalculation.ts)

核心是一个 `computed` 属性，输入 PlanData，输出 60 个月的计算结果数组：

```typescript
interface MonthResult {
  month: number           // YYYYMM
  incomeItems: { name: string; amount: number }[]
  expenseItems: { name: string; amount: number }[]
  totalIncome: number
  totalExpense: number
  investReturn: number    // 理财收益
  netSavings: number      // 净储蓄
  cumSavings: number      // 累计储蓄
  isAnchor: boolean       // 是否为锚点月
}
```

计算流程（逐月递推）：
1. 遍历规划期内每个月
2. 对每个现金流项目，查找该月生效的金额段（最后一个覆盖的段优先）
3. 汇总收入/支出
4. 计算理财收益 = 上月累计储蓄 × 年利率 / 12
5. 净储蓄 = 收入 − 支出 + 理财收益
6. 累计储蓄 = 上月累计 + 净储蓄，若有锚点则直接取锚点值

### 4.2 网格滑动交互 (TimeGrid.vue)

- 布局：5 行 × 12 列（每年一行，每月一格），从 startMonth 起高亮可选区域
- 交互：mousedown 开始选择 → mousemove 扩展选区 → mouseup 确认
- 输出：{ startMonth, endMonth } 元组
- 视觉：选中区域高亮色，hover 时预览

### 4.3 参数面板 (ParamPanel.vue)

- 顶部：系统参数（当前储蓄、起始月、年化收益率）
- 主体：现金流项目列表（可增删）
  - 每个项目：名称、类型切换、金额段列表
  - 每个金额段：金额输入 + TimeGrid 选时间范围
- 底部：月度锚点管理

### 4.4 年度汇总表 (AnnualTable.vue)

按年聚合 MonthResult：
- 年初储蓄、各项收入逐行、收入小计
- 各项支出逐行、支出小计
- 年度结余、年末储蓄

### 4.5 月度明细表 (MonthlyTable.vue)

每行一个月（共60行），列为固定列 + 动态列：
- 固定列：年份、月份
- 动态列：每个收入项目、每个支出项目（根据用户添加的 CashFlowItem 动态生成）
- 汇总列：理财收益、总收入、总支出、净储蓄、累计储蓄
- 列头 sticky 固定，表格纵向滚动时始终可见
- 年份行可作为分组依据，视觉上区分不同年份

  - 单项收入场景

  - 锚点覆盖逻辑

- 实现 useCalculation.ts

## 7. 验证方式

- `npm run test` 运行全部单元测试（Vitest）
- `npm run dev` 启动开发服务器
- 浏览器打开，添加几个收支项目，验证：
  1. 实时计算正确（修改金额立即反映到表格）
  2. 时间段选择可用（网格滑动）
  3. 锚点功能正常（设置后累计储蓄从锚点重新计算）
  4. 刷新后数据不丢失（localStorage）
  5. 数字格式化正确（千分位、负数红色）
