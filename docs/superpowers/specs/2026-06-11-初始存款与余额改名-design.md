# 初始存款与「余额」改名为「存款」设计

> 日期：2026-06-11
> 状态：已确认，待实现

## 背景

家庭财务规划应用当前累计储蓄从 0 起步（`useCalculation.ts` 首月 `prevCum = 0`），用户若已有初始积蓄，只能借首月锚点模拟，体验不佳且语义混淆。同时 UI 各处用「余额」描述累计值，用户希望统一为「存款」。

## 目标

1. 新增「初始存款」字段，作为规划起点本金，从首月起参与投资收益。
2. 将所有用户可见的「余额」文案统一改为「存款」。

## 需求澄清结论

- **初始存款语义**：作为起点本金，**参与投资收益**。首月「上月累计」= 初始存款，理财收益 = 初始存款 × 年利率 / 12。
- **改名范围**：全部用户可见文案统一替换「余额」→「存款」。内部变量名（`cumSavings`、`actualSavings`、`monthlyBalance` 等）保持不变。

## 方案对比

- **A（选定）：`SystemParams` 新增 `initialDeposit`，计算首月起点 = 初始存款。** 与 `startMonth`/`annualRate` 同属起始配置，改动集中、语义清晰。
- **B：用首月特殊锚点实现** —— 锚点语义是「覆盖累计值」，与「起点本金」混淆，且无法参与投资收益，排除。
- **C：用一列现金流表示** —— 初始存款不是月度收支流，放进现金流列语义错误，排除。

## 选定设计（方案 A）

### 1. 数据模型（src/types.ts）

`SystemParams` 新增字段：

```ts
export interface SystemParams {
  startMonth: number
  annualRate: number
  initialDeposit: number   // 新增：初始存款（元），作为累计计算的起点本金
}
```

### 2. 计算逻辑（src/composables/useCalculation.ts）

`calculate` 中首月 `prevCum` 由 `0` 改为初始存款：

```ts
// 首月 prevCum = 初始存款（起点本金，参与投资收益）
const prevCum = index === 0
  ? plan.systemParams.initialDeposit
  : results[index - 1].cumSavings
```

效果：
- 首月理财收益 = 初始存款 × 年利率 / 12
- 首月月末存款 = 初始存款 + 当月结余（当月结余含上述理财收益）
- 后续月份不变，天然继承
- 锚点仍优先覆盖某月累计值（首月若设锚点，锚点优先——边缘情况，保持现状）

无需改动的连带逻辑：
- 年度表 `getStartSavings`：首年年初值 = 首月 `cumSavings − 当月结余`，代入后正好 = 初始存款，自动正确。
- 公式弹窗「存款 = 上月存款 + 当月结余」：首月「上月存款」隐含为初始存款，公式文字成立。

同步更新 `useCalculation.ts` 相关代码注释（如「首月 prevCum = 0…」「计算月末余额」），使注释与新逻辑、新文案一致。

### 3. 状态管理与持久化（src/composables/useStore.ts）

- `createDefault`：`systemParams.initialDeposit = 0`
- `normalizeWorkspace`：对每个 scenario，若 `systemParams.initialDeposit` 缺失或非有限数，补 `0`（轻量容错，保留开发中已有方案数据；不做复杂迁移）。
- `isValidPlanData`：`initialDeposit` 缺失视为可接受（由 normalize 补全）；存在时需为有限数。

### 4. UI

**输入框（src/App.vue）**：header 中「起始月份」「年化收益率」旁新增「初始存款」输入框，双向绑定 `data.systemParams.initialDeposit`：

```vue
<div class="flex items-center gap-2">
  <label class="text-xs whitespace-nowrap">初始存款</label>
  <input
    v-model.number="data.systemParams.initialDeposit"
    type="number"
    class="border rounded px-2 py-1 text-sm w-28"
  />
</div>
```

**改名（仅用户可见文案，内部变量名不动）**：

| 位置 | 文件 | 原 | 改 |
|---|---|---|---|
| 月度表列名 / 公式标签 | MonthlyTable.vue | 余额 | 存款 |
| 月度表表头 | MonthlyTable.vue | 余额 | 存款 |
| 月度表 aria-label | MonthlyTable.vue | 编辑 X 月末余额 | 编辑 X 月末存款 |
| 公式弹窗字段标签 | FormulaPopover.vue | 余额 | 存款 |
| 公式弹窗锚点公式 | FormulaPopover.vue | 锚点月份，余额 = … | 锚点月份，存款 = … |
| 公式弹窗累计公式 | FormulaPopover.vue | 余额 = 上月余额 + 当月结余(…) | 存款 = 上月存款 + 当月结余(…) |
| 年度表行名 | AnnualTable.vue | 年初余额 / 年末余额 | 年初存款 / 年末存款 |

### 5. 测试

**更新现有断言**：将相关测试中「余额」「年初余额」「年末余额」「上月余额」相应改为「存款」「年初存款」「年末存款」「上月存款」：
- tests/components/MonthlyTable.spec.ts
- tests/components/AnnualTable.spec.ts
- tests/components/FormulaPopover.spec.ts

**新增初始存款测试**（tests/composables/useCalculation.spec.ts、tests/composables/useStore.spec.ts）：
- 首月 `cumSavings` = 初始存款 + 当月结余
- 首月 `investReturn` = 初始存款 × 年利率 / 12
- `initialDeposit = 0` 时行为与现状一致（回归）
- 修改初始存款后重新计算结果随之变化
- 年度表首年「年初存款」= 初始存款
- useStore：默认 `initialDeposit = 0`；缺失字段时 normalize 补 0
