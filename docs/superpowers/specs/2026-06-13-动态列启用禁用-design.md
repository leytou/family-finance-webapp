# 动态列启用/禁用（不删除即排除统计）

## 目标

月度表的现金流动态列支持「启用 / 禁用」切换。禁用后，该列的数值照常显示（灰显），但**不计入任何统计**（月度收入/支出/结余/存款、年度聚合）。用户无需删除列即可临时排除某项，随时点回启用。典型场景：临时关掉「旅游」「换车」等可选支出，看基础储蓄曲线。

## 背景

「动态列」即 `plan.columns`（`FlowColumn[]`），是用户增删改的收入/支出项。统计链路：

- `useCalculation.ts` 的 `calculate()` 遍历 `plan.columns`，逐列 `resolveColumnValue` 解析当月值，汇总为 `totalFlow` / `monthlyIncome` / `monthlyExpense`，进而算出结余与累计存款 `cumSavings`。
- `MonthlyTable.vue` 列头/单元格直接遍历 `store.data.value.columns` 渲染；单元格值经 `getColumnValue(result, column.id)` 从 `result.columnValues` 取。
- `AnnualTable.vue` 只接收 `results`，靠 `result.columnValues` 按列名做年度聚合（`sumColumnValues`），**无法访问列定义**。
- 快照封存（`addSnapshot`）把当前 `cumSavings` 序列冻结；计划对比（`buildComparison`）基于 `cumSavings`。

因此「禁用」必须在 `calculate` 排除、又要让月表灰显禁用列的数值、还要让年表自动跳过——三处需协调。

## 架构选择

- **方案 A（采用）**：禁用列**仍留在 `columnValues`**（保留解析值供灰显），由 `enabled` 标记区分；统计时只汇总启用列。
- 方案 B（否决）：禁用列从 `calculate` 彻底剔除 —— 月表灰显得单独重算每列值，逻辑分裂，否决。

用户已确认「禁用列数值灰显保留」，自然落到方案 A。

## 范围

- 新增 `FlowColumn.enabled` 持久化字段；新增 `setColumnEnabled` 状态操作。
- `calculate` 汇总时跳过禁用列；`AnnualTable` 聚合时跳过禁用列。
- 月表列头加 hover 切换图标、禁用列灰显 + 名称删除线。
- 旧数据（无 `enabled` 字段）按启用加载，向后安全。
- **不改**固定列、锚点、快照封存/对比的计算逻辑（它们读取的 `cumSavings` 已自动反映禁用状态）。

## 数据流

```
点击列头 hover 出现的 👁 图标
  → store.setColumnEnabled(id, !enabled)
  → FlowColumn.enabled 更新
  → 现有 watch(deep) 触发 localStorage 自动保存（300ms 防抖）
  → calculate 重算：禁用列仍在 columnValues，但不进 totalFlow/income/expense
  → cumSavings / 结余 随之变化 → 月表统计列 + 年表 + 对比列联动刷新
  → 列头/单元格渲染：禁用列名称加删除线、整列 opacity 灰显
```

## 详细设计

### 1. `types.ts`

```ts
export interface FlowColumn {
  id: string
  name: string
  entries: Record<number, number>
  yearlyMonths?: Record<number, true>
  enabled?: boolean          // 新增：缺省（undefined）= 启用；false = 禁用（不计入统计）
}

export interface MonthResult {
  month: number
  // 每项新增可选 enabled，供年表过滤。设为可选（而非必填）：测试夹具与旧数据
  // 常不带该字段，运行时统一用 `enabled !== false` 把缺省视为启用。
  // resolveColumnValue 的返回类型仍是 enabled: boolean（恒产，可赋值给可选字段）。
  columnValues: { id: string; name: string; amount: number; isEdited: boolean; enabled?: boolean }[]
  // ...其余字段不变
}
```

### 2. `useCalculation.ts`

`resolveColumnValue` 的返回对象补 `enabled`（顶部算一次，三个 return 分支都带上）：

```ts
export function resolveColumnValue(column, month) {
  const enabled = column.enabled !== false   // undefined / true → 启用；仅 false → 禁用
  // ...原逻辑不变，每个 return 增加 enabled 字段
  // 例如规则1：
  return { id: column.id, name: column.name, amount: column.entries[month], isEdited: true, enabled }
}
```

`calculate` 汇总只取启用列；`columnValues` 仍含全部列：

```ts
const columnValues = plan.columns.map(col => resolveColumnValue(col, month))

// 仅启用列参与汇总；禁用列保留在 columnValues 供灰显，但不计入
const active = columnValues.filter(cv => cv.enabled !== false)   // 缺省(undefined)视为启用
const totalFlow = active.reduce((sum, col) => sum + col.amount, 0)
const monthlyIncome = active.reduce((sum, col) => col.amount > 0 ? sum + col.amount : sum, 0)
const monthlyExpense = active.reduce((sum, col) => col.amount < 0 ? sum + Math.abs(col.amount) : sum, 0)
// investReturn / monthlyBalance / cumSavings / push 结果不变
```

### 3. `AnnualTable.vue`

聚合前过滤掉禁用列（禁用列既不单独成行，也不计入年度总额）：

```ts
// yearSummaries 内：
const columnSummaries = sumColumnValues(
  sortedMonths.flatMap(r => r.columnValues).filter(cv => cv.enabled !== false)   // 新增 .filter；缺省视为启用
)
```

### 4. `useStore.ts`

新增 `setColumnEnabled`，并在校验里容错 `enabled`：

```ts
function setColumnEnabled(id: string, enabled: boolean): void {
  const plan = getActivePlan()
  const column = plan.columns.find(col => col.id === id)
  if (!column) return
  column.enabled = enabled
}
```

`isValidColumn` 增加一行（位于现有字段校验后）：

```ts
if (value.enabled !== undefined && typeof value.enabled !== 'boolean') return false
```

`addColumn` 不写 `enabled` 字段（即默认启用），无需改动。返回值导出 `setColumnEnabled`。

### 5. `MonthlyTable.vue`

辅助函数：

```ts
function isColumnEnabled(column: FlowColumn): boolean {
  return column.enabled !== false      // undefined / true → 启用；仅 false → 禁用
}
function toggleColumnEnabled(column: FlowColumn): void {
  store.setColumnEnabled(column.id, !isColumnEnabled(column))   // 取反切换
}
```

列头显示结构改造（原 `v-else` 分支）——名称**左侧**加 hover 👁，禁用列名称删除线：

```html
<span v-else class="flex items-center justify-end w-full">
  <!-- 左侧：启用/禁用切换（hover 显示，与右侧 × 一致） -->
  <button
    type="button"
    class="mr-1 text-gray-400 opacity-0 group-hover:opacity-100 hover:text-gray-700"
    :aria-label="isColumnEnabled(column) ? '禁用此列' : '启用此列'"
    @click="toggleColumnEnabled(column)"
  >👁</button>
  <!-- 列名：禁用时加删除线 + 灰色 -->
  <span
    class="cursor-pointer"
    :class="{ 'line-through text-gray-400': !isColumnEnabled(column) }"
    @dblclick="startRename(column.id, column.name)"
  >{{ column.name }}</span>
  <!-- 右侧：删除按钮（原有，不变） -->
  <button class="ml-1 text-red-600 opacity-0 group-hover:opacity-100 hover:text-red-800"
          @click="confirmRemoveColumn(column.id, column.name)">×</button>
</span>
```

动态列单元格 `<td>` 加禁用灰显 class（**不加删除线**，整列统一半透明）：

```html
<td
  v-for="column in columns"
  :class="[
    getValueClass(getColumnValue(result, column.id).amount),
    { 'bg-blue-100': getColumnValue(result, column.id).isEdited },
    { 'opacity-40': !isColumnEnabled(column) },     // 新增：禁用列灰显
  ]"
>
```

`opacity-40` 作用于整个 `<td>`（含蓝色已编辑底色与文字），禁用态得到统一灰色外观，避免蓝/灰背景冲突。

## 交互冲突处理

| 现有交互 | 是否冲突 | 处理 |
|---|---|---|
| 双击列头重命名 | 否 | 👁 是 click、重命名是 dblclick，互不干扰；重命名 input 显示时 👁/× 仍按 `v-else` 隐藏（input 走 `v-if` 分支） |
| hover × 删除 | 否 | 👁 与 × 同为 `group-hover:opacity-100`，并列显示，各管各的 click |
| 拖拽重排列 | 否 | 禁用列 `:draggable` 仍 true（重命名态才 false），可正常拖动；启用态不影响拖拽 |
| 右键单元格菜单 | 否 | 禁用列单元格仍可右键「同步到下方每年此月」「清除下方编辑值」（这些操作改的是 entries，与 enabled 无关） |
| 蓝色「已编辑」底色 | 视觉 | 禁用时 `opacity-40` 覆盖整个 td，蓝底同步变灰，统一灰显 |

## 边界与错误处理

- **0 列**：无动态列，无影响。
- **全部禁用**：`totalFlow`/`income`/`expense` 每月为 0；`cumSavings = 初始存款 + 累计理财收益`，合法。
- **旧数据**：`enabled` 缺省 → `enabled !== false` 为 true → 启用；`isValidColumn` 不拒绝无该字段的列。
- **数据完整性**：启用/禁用只改 `enabled` 布尔，`entries`/`yearlyMonths` 稀疏数据原样保留；重新启用后数值与统计立即恢复。
- **快照**：封存时 `cumSavings` 已按当时禁用状态算并冻结；之后切换启停不影响已封存快照（符合预期）。
- **计划对比**：基于 `cumSavings`，自动反映禁用状态，无需改动。
- **锚点**：锚点作用于余额列（非 FlowColumn），禁用现金流列不触及锚点逻辑。

## 测试方案（Vitest）

### `tests/composables/useStore.spec.ts` — 新增 `setColumnEnabled` 组

- 禁用后 `column.enabled === false`；启用后恢复为 `true`。
- 不存在的 id：无操作。
- `addColumn` 新列 `enabled` 为 undefined（默认启用）。
- save → reload：`enabled` 状态保留。
- 旧数据（无 `enabled`）加载后按启用，`isValidColumn` 通过。

### `tests/composables/useCalculation.spec.ts` — 核心

- 单列禁用：该列仍在 `columnValues` 且 `amount` 正确，但 `totalFlow`/`monthlyIncome`/`monthlyExpense` 不含其值。
- 全部禁用：三项汇总为 0，`cumSavings` 仅含初始存款 + 理财收益。
- 启用列正常计入（回归）。
- `columnValues` 中每项 `enabled` 字段与列状态一致。

### `tests/components/AnnualTable.spec.ts` — 聚合

- 禁用列不出现在年度表行、不计入年度总额。
- 启用列正常聚合（回归）。

### `tests/components/MonthlyTable.spec.ts` — 渲染与交互

- 禁用列 `<th>` 名称含删除线 class、单元格含 `opacity-40`。
- 启用列无上述 class。
- 点击 👁 切换 `store.data.value.columns[i].enabled`。

## 非目标（YAGNI）

- **批量启用/禁用**：逐列切换即可，不做全选/分组。
- **禁用列单独折叠区**：禁用列保留在原位灰显，不收纳到独立列表（已在澄清环节确认）。
- **禁用状态影响导出**：导出的是完整 Workspace，禁用列照常导出，无需特殊处理。
- **禁用列与拖拽排序的交互强化**：禁用列可正常拖动，不强制排到末尾。
