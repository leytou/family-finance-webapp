# 计划快照偏差对比 — 设计文档

## 1. 背景与目标

用户每月更新一次财务规划。每次更新都会改动现金流列、调整对未来的预测，于是**上一版计划的预测被覆盖**——无法回看「当时我预计能存到多少」，也就无法和实际情况做对比。

现有模型已经具备「实际」数据：`anchors`（每月录入的实际累计储蓄 `actualSavings`）。缺的是「当时的计划」。

本设计引入**计划快照**：用户在更新计划前手动「封存」一份当前的计划累计储蓄曲线（带日期），永久保留、互不覆盖。月度表可选中任一历史快照，逐月并排展示「当时预计 / 实际 / 差额」，看出从哪个月开始偏离、偏多少。

## 2. 核心概念与术语

- **计划快照（Snapshot）**：某个时点封存的「往后每个月的计划累计储蓄」曲线。一旦封存即冻结，后续更新计划不影响已有快照。
- **实际/当前累计**：月度表本就有的累计储蓄列（`MonthResult.cumSavings`）。对已录入 anchor 的月份是真实值；对未来月份是当前最新计划的推算值。
- **差额** = `实际/当前累计 − 当时预计`。正数表示比当时预计的好，负数表示没达到。
- **偏差仅对已发生月份有效**：只有已录入 anchor 的月份才显示差额；未录入 anchor 的（未来）月份差额留空，避免把「后来改了计划」误读成「实际偏差」。

## 3. 数据结构

新增快照类型，挂在 `PlanData` 下（与 `anchors` 平级）：

```ts
interface PlanSnapshot {
  id: string                        // 唯一 id
  name: string                      // 默认按封存时所在月份命名，如「2026-01 计划」，可重命名
  createdMonth: number              // 封存时的系统当前月（YYYYMM），用于排序/展示
  projection: Record<number, number> // 稀疏存储：key=YYYYMM，value=该月计划累计储蓄
}

interface PlanData {
  version: number
  systemParams: SystemParams
  columns: FlowColumn[]
  anchors: MonthlyAnchor[]
  snapshots: PlanSnapshot[]   // 新增
}
```

要点：
- `projection` 直接存封存当下 `calculate()` 算出的各月 `cumSavings`，**值已经是最终累计数**，回看时无需重算，保证「当时预计」永远冻结。
- 快照只存累计储蓄一条曲线（方案 A 的范围），不存现金流明细。
- localStorage 沿用现有整份 `PlanData` 的 300ms 防抖保存。

### 数据迁移

不需要兼容旧数据（项目约定）。但为避免读取已有 localStorage 时 `snapshots` 为 `undefined`，加载校验时将缺失的 `snapshots` 视为 `[]`：`isValidPlanData` 中 `snapshots` 缺失则补空数组，存在则校验每项合法（`id`/`name` 为字符串、`createdMonth` 为整数、`projection` 各键为整数月份、值为有限数）。

## 4. 封存逻辑（useStore 新增）

新增方法：

- `addSnapshot(): PlanSnapshot`
  - 跑一遍当前 `calculate(data.value)`，把每个 `MonthResult` 的 `{ month → cumSavings }` 收进 `projection`。
  - `createdMonth = systemParams.startMonth`（系统当前起始月即「现在」）。
  - `name` 默认 `"${YYYY}-${MM} 计划"`（由 `createdMonth` 格式化）。
  - 推入 `data.value.snapshots`，返回新快照。
- `removeSnapshot(id: string)`：删除指定快照。
- `renameSnapshot(id: string, name: string)`：重命名。

`reset` 时一并清空 `snapshots`（属于当前计划数据的一部分）。

## 5. 计算逻辑（useCalculation 扩展）

新增一个纯函数，把选中快照叠加到月度结果上，不改动核心 `calculate()`：

```ts
interface SnapshotComparison {
  month: number
  predicted: number | null   // 该月当时预计（快照里没有该月则 null）
  actual: number             // 实际/当前累计（= MonthResult.cumSavings）
  diff: number | null        // actual - predicted；仅当该月 isAnchor 且 predicted 非空时有值，否则 null
}

function buildComparison(
  results: MonthResult[],
  snapshot: PlanSnapshot | null,
): SnapshotComparison[]
```

规则：
- `predicted`：`snapshot.projection[month]`，无则 `null`。
- `actual`：对应 `MonthResult.cumSavings`。
- `diff`：仅当**该月 `isAnchor === true`** 且 `predicted !== null` 时 = `actual - predicted`；否则 `null`（未发生月份留空）。
- `snapshot` 为 `null`（未选中任何快照）时，返回全 `null` 的 predicted/diff，月度表照常只显示原有列。

## 6. UI — 月度表内嵌对比列（MonthlyTable）

### 顶部快照选择器
月度表上方加一个下拉：`无 / 2026-01 计划 / 2026-07 计划 …`（按 `createdMonth` 倒序，最新在前）。默认「无」。旁边放：
- **「封存当前计划」按钮**：点击调用 `addSnapshot()`，封存后自动选中这份新快照并进入名称行内编辑态（复用现有列重命名交互）。
- 选中某快照时，名称旁出现**重命名 / 删除**小入口（删除前 confirm 一次）。

### 对比列
选中快照后，月度表右侧追加两列（紧邻现有「累计」列之后）：

| 列 | 内容 |
|---|---|
| 当时预计 | `predicted`，用 `format.ts` 格式化；无值显示 `—` |
| 差额 | `diff`，正数绿色、负数红色、0 中性；`null` 显示空白 |

选「无」时这两列隐藏，月度表回到当前形态，不占空间。

### 交互要点
- 切换快照实时重算对比列（`buildComparison` 很轻）。
- 对比列只读，不可编辑（编辑计划仍在原有现金流列进行，职责不混）。
- 封存动作不修改任何现金流/anchor，纯追加快照，可安全多次封存。

## 7. 计算示例

设系统从 202601 起，年化 2.5%。

**1月封存**，快照「2026-01 计划」projection：

```
202601→5万 202602→7万 202603→9万 202604→11万 202605→13万 202606→15万
```

随后录入实际 anchor：`202603 实际 8.5万`、`202606 实际 13万`。月度表选「2026-01 计划」：

| 月份 | 实际/当前累计 | 当时预计 | 差额 |
|---|---|---|---|
| 202601 | 5 | 5 | 0 |
| 202602 | 7 | 7 | （留空，无 anchor） |
| 202603 | 8.5(anchor) | 9 | −0.5 |
| 202604 | 10.5(推算) | 11 | （留空） |
| 202605 | 12.5(推算) | 13 | （留空） |
| 202606 | 13(anchor) | 15 | −2 |

**7月做下次更新**：先「封存」得快照「2026-07 计划」（起点已贴近现实），老快照原封保留。选「2026-07 计划」时只显示 7 月起的对比，目前 7 月实际略超预计 → 差额 +0.5。两份快照可分别回看「最初计划偏多少」「最近一版准不准」。

## 8. 不在本期范围（YAGNI）

- 曲线图/折线叠加可视化（纯偏差列已够）。
- 自动按月封存（仅手动封存）。
- 现金流单项的计划 vs 实际下钻（本期只做累计储蓄维度）。
- 跨快照互相对比（只做「某快照 vs 实际/当前」）。

## 9. 测试要点

- `addSnapshot`：projection 正确收录各月 `cumSavings`；命名按 `createdMonth` 生成；多次封存互相独立。
- `removeSnapshot` / `renameSnapshot` 正确性。
- `buildComparison`：
  - `predicted` 缺失月返回 `null`；
  - `diff` 仅在 `isAnchor` 且 `predicted` 非空时计算，其余为 `null`；
  - `actual` 对齐 `MonthResult.cumSavings`（含 anchor 覆盖与推算两种情况）；
  - `snapshot` 为 `null` 时全部留空。
- 数据校验：`snapshots` 缺失补 `[]`；非法快照项被拒。
- `reset` 清空 `snapshots`。
