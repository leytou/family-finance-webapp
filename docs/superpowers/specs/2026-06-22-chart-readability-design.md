# 图表易读性优化:拆分上下两块,分清主次

日期:2026-06-22

## 背景

当前「图表」视图(`FinanceChart`)把收入、支出、存款、公积金余额四个系列挤在一张双 Y 轴图里。信息虽全,但「一眼看懂」存在明显障碍(已与项目负责人对齐):

1. **双 Y 轴未标注**:左右两根金额轴(都显示「万」)没标明各自对应哪些系列,看图时分不清某条线该读哪边的刻度。
2. **存款面积压住收支柱**:存款折线下方的渐变面积填充覆盖在收入/支出柱顶上,柱子矮时顶部被糊住,看不出真实高度。
3. **四系列 + 双轴信息过载**:存款(手里的现金)与公积金余额(另一个账户)共用右轴,语义不同却叠在一起,容易让人以为它俩该相加或该比。
4. **折线无数值标记**:存款、公积金为平滑曲线且无数据点/数值,精确值只能靠鼠标悬停。
5. **按月横轴密集且年份重复**:5 年多共 67 个月类目,标签挤;现格式「26/06」把年份每月重复一遍。

## 目标

把综合图按「累计的钱」和「进出的钱」拆成上下两块,共享同一条时间轴(上下对齐),分清主次:

- **上块 · 存款/余额趋势(主角)**:存款线 + 公积金余额线,单金额轴,关键点直接标数值。
- **下块 · 收支结构**:收入柱 + 支出柱,单金额轴,柱顶按需标数值。
- 从结构上消除「双轴混淆」与「面积遮挡」;关键数值直接可读,减少对悬停的依赖。

## 非目标(明确不做)

- **不动计算层**:`useCalculation`、`aggregateByYear` 一行不改;`MonthResult` 字段不变。
- **不动表格视图**:`MonthlyTable`、`AnnualTable`、`KeyMetricsBar` 不受影响。
- **不新增/不删数据系列**:仍是 收入 / 支出 / 存款 / 公积金余额 四个量,只是重新组织布局。
- **不动「按月/按年」切换入口与语义**。
- **不引入新 echarts 依赖**:仍在现有按需注册范围内;若需 `MarkLine`/`LabelLayout` 等组件能力,在实现时按需补注册。

## 用户可见变化

1. 图表视图由「单图」变为「上下两块」,共享时间轴、上下对齐。
2. **上块**:存款折线(主色、加粗、保留渐变面积)+ 公积金余额折线(琥珀副色);只有一根金额轴,标注「余额」;关键点标数值——按月标起点/最低点/终点,按年每个点都标。
3. **下块**:收入柱(朱砂)+ 支出柱(竹青);只有一根金额轴,标注「金额」;按年柱顶直接标数值,按月点太密则靠悬停。
4. **按月横轴分两层**:上层显示月份(1–12 循环,不带年份),下层在每年开始处标年份(2026、2027…),年份交界处拉一条淡色竖线分隔。按年视图横轴仍为单层年份。
5. **悬停联动**:鼠标指某月/某年时,上下两块同时高亮同一时间点(便于对照「这个月收支 → 存款怎么变」)。
6. **标题区**保留「存款 ¥xx.x万」高亮;「按月/按年」切换作用于两块。

## 设计决策与权衡

### 单实例双 grid(采用)vs 两个 echarts 实例

- **单实例 + 双 grid + axisPointer link**:一个 echarts 实例里布置上下两个 grid,各自一组 xAxis/yAxis,series 用 `xAxisIndex/yAxisIndex` 绑定到对应 grid。上下时间轴天然对齐,`axisPointer.link` 即可实现悬停联动。**采用此方案**。
- 权衡:配置比单 grid 复杂,但联动/对齐是「免费」附带的;两个独立实例反而要手写联动,更脆。

### 面积填充保留

上块只有折线、没有柱,存款面积不再遮挡任何东西,保留渐变面积以增强趋势感。

### 公积金余额 0 值

某方案若公积金被提取完,余额变 0,折线贴底——保留真实情况,不做特殊隐藏。

### 数值标注策略(避免按月过载)

- 上块折线:按月只标「起点 / 最低点 / 终点」三个关键点(信息价值最高、不拥挤);按年点少,每个点都标。
- 下块柱:按年柱顶全标;按月不标(靠 tooltip),避免 67 个柱顶文字打架。

### 双层横轴实现

下块 xAxis 的 `axisLabel` 显示月份;用一个 `offset` 下沉的辅助 xAxis(或 `graphic`)在每年首月位置标年份,并用 `splitLine` 在年份边界画淡色竖线。具体用「辅助轴」还是「graphic」在实现阶段以视觉清爽为准定。

## 涉及改动(文件级)

### 计算/数据层(基本不动)

- `src/utils/financeChart.ts` 的 `ChartData`、`buildChartData`、对 `aggregateByYear` 的调用:**语义不变**。`ChartData` 仍提供 `categories / income / expense / cumSavings / fundBalance`。

### 图表 option 构建(`src/utils/financeChart.ts`)

- `buildChartOption` 重构为面向「双 grid」的 option:
  - 拆出上块系列(存款线 + 公积金线)、下块系列(收入柱 + 支出柱)的构建逻辑;
  - `grid` 由单个改为两个(上下排列,共用 xAxis 类目);
  - `xAxis`:下块显示月份的轴 + 辅助年份轴(按月);按年仍单层;
  - `yAxis`:由左右双轴改为两个单轴(上块「余额」轴、下块「金额」轴),各自标注名称;
  - series 增加 `label`(数值标注,按上述策略开关);
  - `tooltip`:`axisPointer.link` 上下联动;formatter 复用现有收入/支出/结余/存款/公积金明细。
- 配色常量(`COLOR_*`)不变。
- 新增/调整纯函数:如「按月找关键点索引(起点/最低/终点)」「年份分组(每年首月索引)」,便于单测。

### 组件(`src/components/FinanceChart.vue`)

- 模板:单个挂载点即可(单实例双 grid 在一个 canvas 里);标题区 + 切换按钮保留。
- `render()` 仍调一次 `setOption`,option 改用新的双 grid 构建。
- `granularity` 切换、`fundEnabled` 变化触发的重渲染逻辑不变。

### 图例

- 单实例下图例点击仍可开关系列,保留该能力;图例项保持「收入/支出/存款/公积金余额」。
