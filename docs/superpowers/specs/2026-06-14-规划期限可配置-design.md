# 规划期限可配置 设计文档

> 日期：2026-06-14
> 状态：待评审

## 1. 背景与现状

当前应用的规划期限固定为 **5 年（60 个月）**，且为**硬编码常量**，分散在 4 处字面量中，互不关联，仅靠注释人工「保持一致」：

| 位置 | 字面量 | 作用 |
|---|---|---|
| `src/composables/useCalculation.ts:4` | `PROJECTION_MONTHS = 60` | 主计算循环边界 |
| `src/composables/useStore.ts:360` | `60` | `syncYearly`「同步到每年此月」循环 |
| `src/composables/useStore.ts:445` | `60` | `syncFundYearly` 公积金年度同步循环 |
| `src/composables/useComparison.ts:27` | `5` | 对比视图年份循环 |

> 行号为探索时快照，实现以实际为准。

`SystemParams` 中**没有**任何期限/年数字段，期限也**未暴露给用户**。用户既不能规划更长（买房/教育/养老），也不能规划更短。

**有利条件**（降低改动风险）：
- `MonthlyTable` / `AnnualTable` / `FinanceChart` 全部跟随 `calculate` 输出长度动态渲染，**不硬编码行数**。
- 所有持久化数据结构（`MonthlyAnchor`、`FundAnchor`、`FlowColumn.entries`、`PlanSnapshot.projection`）均为**按月稀疏存储**，天然兼容任意期限。
- 唯一硬编码年数的展示层是 `ComparisonView.vue`。

**最大隐患**：`useStore.ts` 的两处 `60` 与 `useCalculation` 的 `PROJECTION_MONTHS` 互不引用。若只改其中一处，「右键同步到每年此月」会与主计算期限错位，产生隐性 bug。

## 2. 目标

让用户能自由设置规划期限（更短或更长，**上限 30 年**），期限由「起始月 + 结束月」动态决定，全应用只剩**单一数据源** `endMonth`。

## 3. 关键决策（已确认）

| 决策点 | 选择 | 备选（未选） |
|---|---|---|
| 设置形态 | **起始月 + 结束月（绝对日期）** | 预设档位 / 年数输入框 / 年+月 |
| 期限上限 | **30 年（360 月）** | 15 年 / 50 年 / 无上限 |
| 缩短期限时已有数据 | **静默保留（隐藏不删）** | 直接丢弃 / 弹窗确认 |
| 改起始月时结束月行为 | **不动（纯绝对语义）** | 自动平移保持期限（需存 projectionMonths） |
| 数据存储字段 | **`endMonth`（YYYYMM 绝对）** | `projectionMonths`（相对月数） |

> 「不动结束月」的代价：想把整个规划窗口整体平移，需分别改起始月与结束月各一次。用户已接受。

## 4. 数据模型

### 4.1 `SystemParams` 新增字段（`src/types.ts`）

```ts
interface SystemParams {
  startMonth: number      // 已有
  annualRate: number      // 已有
  initialDeposit: number  // 已有
  endMonth: number        // 新增：规划结束月（YYYYMM，绝对）
}
```

期限不再有常量，全程动态计算（**局部计算值，非存储字段**）：

```
projectionMonths = monthDiff(startMonth, endMonth) + 1
```

### 4.2 新增工具函数（`src/utils/month.ts`）

```ts
// 返回 end 比 start 晚多少个月；end == start 返回 0；含跨年进位
function monthDiff(start: number, end: number): number
```

### 4.3 默认值与迁移

- 新建方案：`endMonth = addMonths(startMonth, 59)`（保持现有 5 年体验）。
- `normalizeWorkspace`：旧数据缺 `endMonth` 时补 `addMonths(startMonth, 59)`。
- `isValidPlanData`：新增 `endMonth` 为合法 YYYYMM 的校验。

## 5. 改动清单

### 5.1 计算与同步逻辑（消灭 4 处字面量）

| 文件:行 | 现状 | 改为 |
|---|---|---|
| `useCalculation.ts:4` | `const PROJECTION_MONTHS = 60` | 删除 |
| `useCalculation.ts:181` | `for index < PROJECTION_MONTHS` | `index < monthDiff(start, end)+1` |
| `useStore.ts:360` `syncYearly` | `for i < 60` | `i < projectionMonths` |
| `useStore.ts:445` `syncFundYearly` | `for i < 60` | `i < projectionMonths` |
| `useComparison.ts:27` | `for year <= 5` | `year <= 年数` |

> `projectionMonths` 在各调用处由 `monthDiff+1` 现算，不引入新存储字段。

`MonthlyTable` / `AnnualTable` / `FinanceChart` **无需改动**（已自适应）。

### 5.2 对比视图动态化（`src/components/ComparisonView.vue:53-64`）

- 第 N 年末行由 `v-for` 按当前期限动态生成（不再写死 5 行）。
- 「5 年总收入/支出/净储蓄」→「**全程**总收入/支出/净储蓄」。
- `useComparison.ts` 的 `ScenarioMetrics.yearEndSavings` 数组长度由固定 5 改为按期限年数生成。
- 顺带消除：期限 <5 年时 `yearEndSavings[4]` 取 `undefined` 的越界 bug。

### 5.3 UI 交互（`src/App.vue` header，起始月右侧）

- 新增结束月选择器，**复用 `MonthPicker`** 组件（与起始月体验一致）。
- 经 `setEndMonth` 受控写入（照搬 `setStartMonth` 模式）。
- 选择器下方提示「**共 X 年 Y 个月**」。

`src/composables/useStore.ts`：新增 `setEndMonth` setter；并增强现有 `setStartMonth`，增加与 `endMonth` 的一致性校验（见 §6）。

## 6. 行为与边界校验

两层防御：

**setter 校验**（`setEndMonth` / 增强的 `setStartMonth`，非法则拒绝写入、不更新 state）；`App.vue` handler 调用 setter 后据其结果显示内联红色提示：
- `endMonth >= startMonth`，否则提示「结束月需晚于起始月」。
- `monthDiff+1 <= 360`，否则提示「规划期限不能超过 30 年」。
- 改 `startMonth` 时同样校验：若导致 `start > end` 或期限超限，拒绝并提示「起始月需早于结束月」/「期限将超 30 年，请先调整结束月」。

**store 层兜底**（`calculate`）：
- 对范围做 `clamp`：`end = clamp(end, start, start+359)`，保证任何异常状态下表格不崩。

**下限**：最短 1 个月（`endMonth == startMonth`）。

**缩短期限**：超范围的锚点/月度编辑/年度同步值**保留在数据中，仅隐藏不参与计算**；延长回来即恢复（数据稀疏存储天然支持）。

## 7. 测试策略（Vitest）

| 目标 | 用例 |
|---|---|
| `monthDiff` | 跨年 / 同年 / 同月 / 非法输入 |
| `calculate` | 期限 = 1月 / 5月 / 360月 输出长度正确；原 60月 回归 |
| `syncYearly` / `syncFundYearly` | ⚠️ **重点**：期限改长后循环覆盖到新范围（防隐蔽 bug） |
| `useComparison` | `yearEndSavings` 长度随期限 |
| `ComparisonView` | 期限 <5 年不越界；期限 >5 年行数正确 |
| store | `setEndMonth`/`setStartMonth` 校验、`normalizeWorkspace` 补默认、`isValidPlanData` 新字段、旧数据加载不丢 |

## 8. 非目标（YAGNI）

- 不做虚拟滚动：30 年 = 360 行，月度表可接受。
- 不做「改起始月自动平移结束月」的智能行为（纯绝对语义）。
- 不做预设档位按钮（结束月选择器已足够）。
- 不兼容旧数据「格式」（仅 `normalize` 补默认值）。
