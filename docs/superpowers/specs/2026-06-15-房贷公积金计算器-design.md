# 房贷月供 & 公积金缴存计算器 设计

> 日期：2026-06-15
> 状态：已定稿，待实现计划
> 基线：当前 master `npx vitest run` = **447 测试全绿（25 文件）**。
> 范围：新增一个独立的「计算器」页面，包含两张表——① 房贷等额本息月供（含总利息、还款总额），② 公积金每月缴存。计算器与现有财务规划数据**完全独立**。

## 1. 背景与范围

现有应用是一套家庭财务规划工具（现金流表格 + 图表 + 多方案对比 + 公积金账户）。规划表里虽已有「房贷月供」「公积金缴存」两栏（手动填数），但用户希望有一个**独立的小计算器**：输入贷款金额和各期限利率，立刻看到等额本息月供；输入缴存基数和比例，立刻看到公积金每月缴存。算完自己参考，不回填规划表。

**目标：**
- 房贷：贷款金额（共享）+ 5/10/15/20/30 年各自年利率 → 每个期限的**月供 / 总利息 / 还款总额**，以表格展示。
- 公积金：缴存基数 + 个人比例 + 单位比例 → 个人 / 单位 / 合计 每月金额，以表格展示。
- 输入记忆：记住上次填的值，**单独存一块**，与规划数据互不混。
- 入口：顶栏新增独立「🧮 计算器」按钮，点击切到计算器页面。

**非目标（YAGNI）：**
- **不与规划表联动**：算出的月供/缴存不回填规划表的「房贷月供」「公积金缴存」列（用户明确选「纯独立」）。
- **不进撤销/重做**：计算器输入改动不影响规划表的撤销/重做历史，两边独立。
- 不做等额本金（只做等额本息）。
- 不做公积金缴存基数上限 / 比例上限校验（填多少算多少）。
- 不做响应式收纳 / 移动端适配。
- 不做「一键填回规划表」「导出计算结果」等附加动作。

## 2. 已确认的交互决策

| 决策点 | 选择 |

| 输入是否记忆 | **记忆**，单独存档位 `family-finance-calculator`，与规划数据分离 |
| 入口位置 | **独立工具按钮**：「表格/图表/对比」三按钮右侧用竖线隔开放「🧮 计算器」，点击切到计算器页面（整块主区切换） |
| 房贷输出列 | **月供 + 总利息 + 还款总额**（三列） |
| 公积金表格结构 | **个人 / 单位 / 合计** 三行，每行含比例 + 每月金额 |
| 页面内布局 | 两张表**左右并排**（房贷左、公积金右） |
| 计算器视图下的参数行 | **隐藏**（起始月份/年化/初始存款/公积金在此视图用不到） |
| 状态管理做法 | **新建独立组合式函数**（仿现有状态管理写法），纯函数计算 + 单独 localStorage |

## 3. 计算公式（核心逻辑，纯函数）

### 3.1 房贷 · 等额本息

每月还款额：
```
月供 = 本金 × 月利率 × (1+月利率)^月数 ÷ ((1+月利率)^月数 − 1)
月利率 = 年利率 ÷ 12
月数   = 年限 × 12
```
派生：
- **还款总额** = 月供 × 月数
- **总利息** = 还款总额 − 本金

**0 利率特例**（避免除零）：年利率为 0 时，`月供 = 本金 ÷ 月数`，`总利息 = 0`，`还款总额 = 本金`。

**验证基准**（本金 1,000,000、年利率 3.5%）：

| 期限 | 月数 | 月供 | 还款总额 | 总利息 |
|---|---|---|---|---|
| 5 年 | 60 | 18,192 | 1,091,505 | 91,505 |
| 10 年 | 120 | 9,889 | 1,186,630 | 186,630 |
| 15 年 | 180 | 7,149 | 1,286,789 | 286,789 |
| 20 年 | 240 | 5,800 | 1,391,903 | 391,903 |
| 30 年 | 360 | 4,490 | 1,616,561 | 616,561 |

### 3.2 公积金缴存

```
个人每月 = 缴存基数 × 个人比例
单位每月 = 缴存基数 × 单位比例
合计    = 个人每月 + 单位每月
（比例合计 = 个人比例 + 单位比例）
```

> **单位约定**：利率、比例均以**百分比**存储（如 3.5 表示 3.5%，12 表示 12%）。纯函数内部统一 `÷ 100` 转小数计算。

## 4. 两张表格 · 列与输入

### 4.1 房贷表

顶部一个共享输入：**贷款金额（元）**。

| 期限 | 年利率(%) | 月供(元) | 总利息(元) | 还款总额(元) |
|---|---|---|---|---|
| 5 年 | _可填_ | 自动 | 自动 | 自动 |
| 10 年 | _可填_ | 自动 | 自动 | 自动 |
| 15 年 | _可填_ | 自动 | 自动 | 自动 |
| 20 年 | _可填_ | 自动 | 自动 | 自动 |
| 30 年 | _可填_ | 自动 | 自动 | 自动 |

- **期限**：固定文字（5 个固定期限常量），不可改。
- **年利率(%)**：每行独立可填，支持小数（3.85 等）；空/负按 0。
- **月供 / 总利息 / 还款总额**：实时算、只读；金额取整 + 千分位（复用 `formatCurrency`）。

### 4.2 公积金表

顶部一个输入：**缴存基数（元）**。

| 项目 | 缴存比例(%) | 每月金额(元) |
|---|---|---|
| 个人 | _可填_ | 自动 |
| 单位 | _可填_ | 自动 |
| **合计** | 自动（比例之和） | 自动（金额之和，高亮加粗） |

- **个人 / 单位 比例**：可填，支持小数（12.5 等）；空/负按 0。
- **每月金额**：基数 × 比例，只读。
- **合计行**：底色高亮（`bg-brand-50`）、加粗，比例与金额均求和。

## 6. 数据结构与存储

### 6.1 状态结构（`CalculatorState`）

```ts
const MORTGAGE_TERMS = [5, 10, 15, 20, 30] as const   // 固定期限

interface CalculatorState {
  loanAmount: number                         // 贷款金额（元）
  rates: Record<typeof MORTGAGE_TERMS[number], number>  // 各期限年利率（百分比）
  fundBase: number                           // 公积金缴存基数（元）
  personalRate: number                       // 个人缴存比例（百分比）
  employerRate: number                       // 单位缴存比例（百分比）
}
```

> 利率 / 比例以**百分比**存储（3.5 = 3.5%、12 = 12%），与输入框直接对应；纯函数内部 ÷100 转小数。

### 6.2 存储与默认

- **存档位**：独立 localStorage key `family-finance-calculator`（与规划数据 `family-finance-plan` 分离）。
- **写入**：状态变化时防抖写入（300ms，仿现有 `useStore` 做法）。
- **读取**：应用启动时读；缺失 / 损坏（JSON 解析失败或字段非法）→ 回退默认值并立即存一份。
- **默认值**：首次无存档时所有字段为 0（`loanAmount: 0`、各 `rates` 为 0、`fundBase: 0`、`personalRate: 0`、`employerRate: 0`），结果实时算。
- **校验**：读取后做字段合法性校验（均为有限数；`rates` 含全部 5 个期限键），非法字段补默认。
- **版本号**：`version: 1`，便于后续结构变更时迁移。

## 7. 边界与输入处理

统一原则：**不弹窗、不报错、不卡输入**；空 / 非法（NaN）/ 负数一律按 0 参与计算，结果实时跟着变。

| 输入 | 处理 |
|---|---|
| 贷款金额 空 / 负 / 非数 | 按 0；月供 = 0 |
| 年利率 空 / 负 / 非数 | 按 0；触发 0 利率特例（月供 = 本金 ÷ 月数、总利息 0） |
| 缴存基数 空 / 负 / 非数 | 按 0 |
| 个人 / 单位比例 空 / 负 / 非数 | 按 0 |
| 利率 / 比例范围 | 不卡上下限，支持小数（3.85、12.5），填多少算多少 |
| 金额显示 | 取整 + 千分位（`formatCurrency`） |

> 输入过程中间态（如输入框临时为空、`"3."`）不得崩计算：解析用 `Number()`，`NaN` 一律按 0。

## 9. 测试计划

1. **计算公式（`tests/utils/calculator.spec.ts`，重点）**：
   - 等额本息月供：本金 100 万、年利率 3.5%，5 个期限的月供 / 总利息 / 还款总额 全部命中第 3.1 节基准表。
   - 0 利率特例：月供 = 本金 ÷ 月数、总利息 = 0、还款总额 = 本金，不抛错。
   - 0 本金：月供 = 0、总利息 = 0、还款总额 = 0。
   - 不变量：`总利息 = 还款总额 − 本金`（多组随机参数）。
   - 公积金：个人 = 基数×个人比例、单位 = 基数×单位比例、合计 = 个人+单位；0 基数 / 0 比例 / 比例>100 均正确。
   - 负数 / NaN 输入按 0（边界）。
2. **状态与存储（`tests/composables/useCalculator.spec.ts`）**：
   - 首次无存档 → 全 0 默认。
   - 改字段 → 写入 `family-finance-calculator`（且**不**写 `family-finance-plan`）。
   - 重新加载 → 读回一致。
   - 存档损坏 / 字段缺失 → 回退默认并重写。
3. **组件（`tests/components/CalculatorView.spec.ts`）**：
   - 改贷款金额 → 5 行月供/利息/总额更新。
   - 只改某行年利率 → 仅该行变化。
   - 改缴存基数 / 个人 / 单位比例 → 个人、单位、合计金额更新；合计 = 两项之和。
   - 空输入不崩、按 0 显示结果。
4. **入口与隔离（`tests/App.spec.ts`）**：

   - 切回表格/图表/对比正常；现有视图切换断言不受影响。
   - 在计算器改输入后撤销/重做**不**改变规划数据（独立）。
