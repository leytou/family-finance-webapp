# 公积金专区列：编辑高亮 + 右键清除菜单

## 背景

月度表（`MonthlyTable.vue`）公积金专区里，**房贷月供**、**公积金缴存**两列现在能点开编辑，但存在两处缺失：

- 编辑（手填）后，单元格**没有**浅蓝色底标记——不像动态现金流列、存款锚点列、公积金月冲列那样，手填后变浅蓝（`bg-brand-50`）。
- **没有**右键菜单，缺少「清除该值」「清除下方编辑值」等单格/批量重置入口。

两类列的存储结构其实是现成的：

- 房贷月供：`fund.mortgage`（`FlowColumn`，`entries` 内部存负数、显示绝对值）。
- 公积金缴存：`fund.contribution`（`FlowColumn`，`entries` 存正数）。

底层 store 的删除方法 `store.updateFundEntry(field, month, null)` 也已存在，与动态列的 `store.updateColumnEntry(...)` 语义一致（删 entry 时联动清除 `yearlyMonths` 标记）。所以本次**纯前端复用**，不改 store、不改计算。

## 需求确认

| 决策点 | 结论 |
| --- | --- |
| 覆盖列 | **房贷月供** + **公积金缴存**两列（公积金月冲列**不动**，保持其 auto/手填联动视觉） |
| 浅蓝底 | 该月手填过值即显示 `bg-brand-50`；清掉手填值后消失 |
| 右键菜单 | 复用现有统一菜单，三项**全部显示** |
| 「同步到下方每年此月」 | 在这两列上**恒为禁用置灰**（显示但不可点；store 的 `syncFundYearly` 虽已存在，本期不对专区列开放） |
| 「清除该值」 | 当前格手填过值时启用，点击删该月 entry |
| 「清除下方编辑值」 | 下方存在手填过的格子时启用，点击批量删 |
| 二次确认 | 不需要 |
| 菜单宽度 / 排版 | 复用现有 `openContextMenu`（`expectedWidth: 192`） |

## 方案

改动**仅限** `MonthlyTable.vue`。`ContextMenu.vue`、`useStore.ts`、`useCalculation.ts`、`types.ts` 均不改。

### 1. 新增 sentinel 与映射辅助函数

与现有 `BALANCE_COLUMN_ID` / `FUND_BALANCE_COLUMN_ID` 同模式，给两列各配一个内部标识，并把标识映射回 fund 字段名：

```ts
// 房贷月供 / 公积金缴存列在右键菜单逻辑中的特殊列标识
const FUND_MORTGAGE_COLUMN_ID = '__fund_mortgage__'
const FUND_CONTRIBUTION_COLUMN_ID = '__fund_contribution__'

// 标识 → fund 字段名；非专区固定列返回 null
function fundFieldFromColumnId(columnId: string): 'mortgage' | 'contribution' | null {
  if (columnId === FUND_MORTGAGE_COLUMN_ID) return 'mortgage'
  if (columnId === FUND_CONTRIBUTION_COLUMN_ID) return 'contribution'
  return null
}

// 房贷月供 / 公积金缴存：该月是否手填过（决定浅蓝底）
function isFundEntryEdited(field: 'mortgage' | 'contribution', month: number): boolean {
  return fund.value ? String(month) in fund.value[field].entries : false
}
```

### 2. 现有右键逻辑增加 fund 分支

在 `editedBelowRows`、`clearEditedBelow`、`isCurrentCellEdited`、`clearCurrentValue` 四处，识别上述两个 sentinel，按 fund 字段读写 `entries`：

```ts
// 返回某列在指定月份"严格下方"所有编辑过值的行
function editedBelowRows(columnId: string, month: number): MonthResult[] {
  const fundField = fundFieldFromColumnId(columnId)
  return props.results.filter(r =>
    r.month > month &&
    (columnId === BALANCE_COLUMN_ID ? r.isAnchor
      : columnId === FUND_BALANCE_COLUMN_ID ? r.isFundAnchor
      : fundField ? isFundEntryEdited(fundField, r.month)
      : getColumnValue(r, columnId).isEdited))
}

function clearEditedBelow(columnId: string, month: number): void {
  const fundField = fundFieldFromColumnId(columnId)
  editedBelowRows(columnId, month).forEach(r => {
    if (columnId === BALANCE_COLUMN_ID) store.removeAnchor(r.month)
    else if (columnId === FUND_BALANCE_COLUMN_ID) store.removeFundAnchor(r.month)
    else if (fundField) store.updateFundEntry(fundField, r.month, null)
    else store.updateColumnEntry(columnId, r.month, null)
  })
}

function isCurrentCellEdited(columnId: string, month: number): boolean {
  if (columnId === BALANCE_COLUMN_ID) {
    return props.results.find(r => r.month === month)?.isAnchor ?? false
  }
  const fundField = fundFieldFromColumnId(columnId)
  if (fundField) return isFundEntryEdited(fundField, month)
  const column = columns.value.find(c => c.id === columnId)
  return column ? String(month) in column.entries : false
}

function clearCurrentValue(columnId: string, month: number): void {
  const fundField = fundFieldFromColumnId(columnId)
  if (columnId === BALANCE_COLUMN_ID) store.removeAnchor(month)
  else if (fundField) store.updateFundEntry(fundField, month, null)
  else store.updateColumnEntry(columnId, month, null)
}
```

### 3. 「同步」项对专区固定列恒置灰

`contextMenuItems` 中，专区固定列的「同步到下方每年此月」恒 `disabled: true`（与动态列按 `hasDirectEntry` 判断区分）：

```ts
const fundField = fundFieldFromColumnId(ctx.columnId)
const isFundFixed = fundField !== null

// 「同步」项：余额列不显示；专区固定列显示但恒置灰；动态列按是否有直接值
if (!isBalanceColumn) {
  const column = columns.value.find(c => c.id === ctx.columnId)
  const hasDirectEntry = column ? String(ctx.month) in column.entries : false
  items.push({
    label: '同步到下方每年此月',
    disabled: isFundFixed ? true : !hasDirectEntry,
    onClick: () => store.syncYearly(ctx.columnId, ctx.month),  // 置灰项永不触发
  })
}
```

> 「清除该值」「清除下方编辑值」两项无需改：`isCurrentCellEdited` / `countEditedBelow` 已能识别专区固定列；「清除下方编辑值」标签逻辑（仅 `FUND_BALANCE_COLUMN_ID` 显示「清除下方公积金锚点」，其余显示「清除下方编辑值」）对专区固定列自然落到后者，正确。

### 4. 模板：浅蓝底 + 右键

房贷月供单元格追加浅蓝底 class 与右键事件（保留原有 `getValueClass`、`border-l-2`）：

```html
<td
  class="px-1 py-0 text-right tabular-nums whitespace-nowrap border-l-2 border-neutral-400 relative"
  :class="[
    getValueClass(-fundMortgageAbs(result.month)),
    { 'bg-brand-50': isFundEntryEdited('mortgage', result.month) },
  ]"
  @contextmenu.prevent="openContextMenu(FUND_MORTGAGE_COLUMN_ID, result.month, $event)"
>
```

公积金缴存单元格同样追加：

```html
<td
  class="px-1 py-0 text-right tabular-nums whitespace-nowrap relative"
  :class="{ 'bg-brand-50': isFundEntryEdited('contribution', result.month) }"
  @contextmenu.prevent="openContextMenu(FUND_CONTRIBUTION_COLUMN_ID, result.month, $event)"
>
```

### 菜单项最终顺序

| 动态现金流列 | 存款余额列 | 公积金余额列 | 房贷月供 / 公积金缴存 |
|---|---|---|---|
| ① 同步到下方每年此月 | ① 清除该值 | ① 清除该值 | ① 同步到下方每年此月 **(置灰)** |
| ② 清除该值 | ② 清除下方编辑值 | ② 清除下方公积金锚点 | ② 清除该值 |
| ③ 清除下方编辑值 | | | ③ 清除下方编辑值 |

专区固定列的菜单形态与动态列一致（三项齐全），仅「同步」项恒不可点。

## 边界与错误处理

- **未编辑格**：「清除该值」`disabled`；判断走 `isFundEntryEdited`（key 存在即编辑过，含值为 0）。
- **房贷月供存负数**：`entries` 内部为负，判断「是否编辑」只看 key 存在性，不受正负号影响；清除用 `updateFundEntry('mortgage', month, null)` 删 key，回退到默认显示（与清空编辑框确认同效）。
- **`yearlyMonths` 联动**：`updateFundEntry(..., null)` 已含联动删除 `yearlyMonths[month]`，专区固定列与动态列行为一致。
- **`fund` 未启用**：专区整块不渲染（外层 `v-if="fund"`），sentinel 与 `isFundEntryEdited` 内不会触发；`isFundEntryEdited` 自身亦兜底 `fund.value` 为空时返回 false。
- **行 hover 覆盖**：现有 `tbody tr:hover td` 规则会让浅蓝在 hover 时变淡绿，与动态列手填格行为一致，不改。
- **与左键编辑共存**：右键打开菜单、左键进编辑框，二者不冲突（左键走 `startEditFundCell`，右键走 `@contextmenu.prevent`）。

## 测试（Vitest，`tests/components/MonthlyTable.spec.ts` 新增）

- 房贷月供：手填某月 → 该格出现 `bg-brand-50`；右键 → 菜单含三项，「同步」项 `disabled`。
- 公积金缴存：同上。
- 「清除该值」：专区列手填格右键点「清除该值」→ `entries` 该月删除、浅蓝消失。
- 「清除下方编辑值」：专区列下方多月手填后，点该项 → 下方 entry 全部清除、浅蓝消失；上方不受影响。
- 「同步」项恒禁用：无论该格是否手填，「同步」项均 `disabled`。

## 不做的事

- 不改 `ContextMenu.vue`、`useStore.ts`、`useCalculation.ts`、`types.ts`。
- 不对专区列开放「同步到下方每年此月」（保持置灰）。
- 不动公积金月冲列（保留其 auto/手填联动视觉，不加右键菜单）。
- 不做二次确认、不做「清除整列 / 清除上方」等其它项。
