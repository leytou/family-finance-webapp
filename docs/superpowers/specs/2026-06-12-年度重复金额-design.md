# 年度重复金额（每年此月）设计

> 日期：2026-06-12
> 状态：已确认，待实现

## 背景

家庭财务规划应用中，现金流列的值由 `resolveColumnValue`（`src/composables/useCalculation.ts`）按**向前延续**规则解析：某月填了值 X，之后所有月份自动延续 X，直到下一个编辑点。

这种「阶梯延续」适合月薪、涨薪后新工资等持续值，但对**每年固定月份发生一次**的脉冲项（年终奖、年付保险、车险、年付物业等）非常笨拙：为了让 X 只在一个月生效，必须在下一个月显式填 0 来打断延续，且下一年又要重复整套操作。60 个月的规划期里要重复多次。

## 目标

让「每年固定月份、金额基本固定」的现金流项一次录入即可覆盖所有年份的同一月份，且中间月份不被该值蔓延，无需逐月填 0。

## 需求澄清结论

- **金额特征**：每年基本固定；个别年份不同时，可单独覆盖那一年。
- **发生月份**：固定（如年终奖总在 12 月）。
- **列归属（假设）**：年度项单独占一列（如「年终奖」一列、「车险」一列）。
- **关键调整——快照式同步**：源头值（如 202612）支持单独修改，**不同步**到其它年份；只有**右键「同步到每年此月」**时才把当前值批量传播到所有年份同月。每个年份同月是独立 entry，而非引用式自动跟随。
- **同步覆盖**：直接覆盖该列所有年份同月的值（含此前单独定制过的），**不弹确认、不提示数量**。

## 方案对比

- **A（选定）：`FlowColumn` 新增 `yearlyMonths` 标记 + 解析时跳过 + 右键手动同步。** `yearlyMonths` 与 `entries` 正交，`entries` 保持纯数字零改动；解析规则只改一行（延续时跳过标记月）；同步是显式右键操作，符合「快照式」语义。
- **B：右键批量复制、不改数据模型。** 批量写入的都是普通 entry，会被现有规则当成延续源，导致年度点之后的月份被蔓延成该值（中间月变 50000 而非 0）。要修正只能给 entry 加「不延续」标记——即退化回方案 A。排除。
- **C：新增 `yearly` 列类型。** 引入列类型概念，每个年度列还要绑定一个固定月份，改动最大、灵活性最低。排除。

## 选定设计（方案 A）

### 1. 数据模型（src/types.ts）

`FlowColumn` 新增一个**可选**字段，`entries` 完全不变：

```ts
export interface FlowColumn {
  id: string
  name: string
  entries: Record<number, number>          // 不变：金额（元）
  yearlyMonths?: Record<number, true>      // 新增：哪些月是「年度重复项」(key=YYYYMM，存在即标记)
}
```

关键决策：yearly 信息独立存放，不并入 `entries`。这样 `updateColumnEntry`、`isValidColumn` 以及所有读写 `entries` 的逻辑零改动；旧数据没有 `yearlyMonths`，`undefined` 即视为「无年度项」，**天然兼容、无需迁移**。

### 2. 解析规则（src/composables/useCalculation.ts）

`resolveColumnValue` 唯一变化：**向前延续时，跳过 `yearlyMonths` 标记的月**。

```ts
// 规则2: 向前查找最近的 entry（key < month），跳过被标记为 yearly 的月
const isYearlyKey = (k: number) => Boolean(column.yearlyMonths?.[k])
const entryKeys = Object.keys(column.entries)
  .map(Number)
  .filter(key => key < month && !isYearlyKey(key))
```

规则 1（该月有直接 entry → 用它）与规则 3（无源 → 0）不变。

验证（年终奖 50000，所有同月均标记 yearly）：

| 月份 | 结果 | 说明 |
|---|---|---|
| 202612 | 50000 | 直接 entry |
| 202701–202711 | **0** | 向前跳过 202612(yearly)，无可用源 → 0 |
| 202712 | 50000 | 直接 entry |

月度列（无 `yearlyMonths` 标记）解析行为**完全不变**，无回归风险。

### 3. 同步与编辑语义（src/composables/useStore.ts）

| 操作 | 行为 |
|---|---|
| 直接编辑单元格 | 只写 `entries[该月]`，不动 `yearlyMonths`，**不波及其它年份** |
| 右键「同步到每年此月」 | 把当前月值复制到投影范围内所有同月（含源头）并打 `yearlyMonths` 标记 |
| 清空单元格 | 删 `entries[该月]`，**联动删除** `yearlyMonths[该月]` |

新增方法 `syncYearly`：

```ts
function syncYearly(colId: string, month: number): void {
  const plan = getActivePlan()
  const column = plan.columns.find(c => c.id === colId)
  if (!column) return
  const amount = column.entries[month]
  if (amount === undefined) return                 // 无值不可同步
  if (!column.yearlyMonths) column.yearlyMonths = {}
  const moy = month % 100
  const start = plan.systemParams.startMonth
  for (let i = 0; i < 60; i++) {                   // 与 calculate 的 PROJECTION_MONTHS 一致
    const m = addMonths(start, i)
    if (m % 100 === moy) {
      column.entries[m] = amount
      column.yearlyMonths[m] = true
    }
  }
}
```

`updateColumnEntry` 在删除分支联动清标记：

```ts
if (value === null) {
  delete column.entries[month]
  if (column.yearlyMonths) delete column.yearlyMonths[month]   // 新增联动
} else {
  column.entries[month] = value
}
```

### 4. UI（src/components/MonthlyTable.vue）

- **右键菜单**新增项 `同步到每年此月`，仅在现金流列且该月**存在直接编辑值**（`entries[month]` 有值）时启用（余额列 `__balance__` 不显示此项）。在 `contextMenuItems` computed 中、现有「清除下方编辑值」之前追加。点击调用 `store.syncYearly(columnId, month)`。
- **角标**：现金流单元格若 `yearlyMonths[month]` 存在，在蓝底（已编辑）基础上叠加 `↻` 角标，提示「改它不影响其它年份」。

### 5. 持久化与校验（src/composables/useStore.ts）

- `isValidColumn`：`yearlyMonths` 为可选字段；存在时校验为对象、键为整数月份。缺失视为合法（旧数据天然通过）。
- `normalizeWorkspace`：无需专门处理（字段可选，缺失即无年度项）。

### 6. 测试（TDD）

**扩展 `tests/composables/useCalculation.spec.ts`（`resolveColumnValue`）**：
- 标记 yearly 的月被延续逻辑跳过 → 其后非同月归 0
- 标记 yearly 的月自身正常返回其 entry 值
- 无 `yearlyMonths` 字段时行为与现状一致（回归）
- 混合：一列中非 yearly entry 仍正常延续，yearly 月不影响其后延续源选取

**新增 `tests/composables/useStore.spec.ts`（`syncYearly` / `updateColumnEntry`）**：
- `syncYearly` 把当前月值写入投影范围内所有同月并打标记，源头也被标记
- `syncYearly` 对无值的月不操作
- 直接修改某月 entry 后，其它年份同月不变（快照语义）
- `updateColumnEntry(null)` 联动清除该月 `yearlyMonths` 标记

**`tests/components/MonthlyTable.spec.ts`**：右键存在直接编辑值的单元格出现「同步到每年此月」、无值时该项禁用；yearly 单元格显示 `↻`。

## 假设与边界

- **假设年度项单独成列**。若同一列混用「月度持续值」与「年度脉冲值」：解析时某月若同时被两者命中，**直接 entry 优先**（yearly 月的金额会覆盖该月，不与月度延续值叠加）。需要叠加时应拆成两列。
- 同步范围为 `startMonth` 起的 60 个月（与 `calculate` 一致）；超出投影范围的月份不涉及。
