# 月度表右键菜单：清除该值（当前单元格）

## 背景

月度表（`MonthlyTable.vue`）现有右键菜单（`ContextMenu.vue`）提供两项：

- 「同步到下方每年此月」（仅现金流列）
- 「清除下方编辑值」（现金流列 + 余额列）

二者作用范围都是「下方」或「批量」。用户希望新增一项「清除该值」，只清除**右键所在当前单元格**的手工编辑值，便于单格快速重置。

两类可编辑单元格的存储：

- **动态现金流列**：编辑值存于 `FlowColumn.entries[month]`，单元格 `isEdited` 标记。
- **余额列**：编辑值存为锚点 `anchors[month]`，单元格 `isAnchor` 标记。

## 需求确认

| 决策点 | 结论 |
| --- | --- |
| 作用对象 | **当前单元格**（当前列 × 当前月），非下方、非整列 |
| 作用列范围 | 动态现金流列 + 余额列 |
| 动态列清除动作 | `store.updateColumnEntry(columnId, month, null)` 删除该月 entry |
| 余额列清除动作 | `store.removeAnchor(month)` 删除该月锚点 |
| 当前格未编辑时 | 菜单项**禁用置灰**（始终显示，不可点） |
| 二次确认 | **不需要**：点击即清除 |
| 菜单名字 | **清除该值** |

## 方案

改动**仅限** `MonthlyTable.vue`。`ContextMenu.vue`、`useStore.ts`、`types.ts` 均不改（通用菜单组件与现有 store 方法已满足）。

### 新增辅助函数

```ts
// 当前格是否已被手动编辑（决定「清除该值」是否可用）
function isCurrentCellEdited(columnId: string, month: number): boolean {
  if (columnId === BALANCE_COLUMN_ID) {
    return props.results.find(r => r.month === month)?.isAnchor ?? false
  }
  const column = columns.value.find(c => c.id === columnId)
  return column ? String(month) in column.entries : false
}

// 清除当前格的编辑值
function clearCurrentValue(columnId: string, month: number): void {
  if (columnId === BALANCE_COLUMN_ID) {
    store.removeAnchor(month)
  } else {
    store.updateColumnEntry(columnId, month, null)
  }
}
```

### 菜单项

在现有 `contextMenuItems` computed 中，于「同步到下方每年此月」**之后**、「清除下方编辑值」**之前**插入：

```ts
items.push({
  label: '清除该值',
  disabled: !isCurrentCellEdited(ctx.columnId, ctx.month),
  onClick: () => clearCurrentValue(ctx.columnId, ctx.month),
})
```

点击菜单项后 `ContextMenu` 自身 `emit('close')` 关闭，无需额外处理。

### 菜单项最终顺序

| 动态现金流列 | 余额列 |
|---|---|
| ① 同步到下方每年此月 | ① 清除该值 |
| ② 清除该值 | ② 清除下方编辑值 |
| ③ 清除下方编辑值 | |

顺序逻辑：先「该格」（同步 / 清除该格），后「下方」批量，由近及远。

## 边界与错误处理

- **未编辑格**：`disabled` 置灰。动态列判断 `String(month) in column.entries`（key 存在即编辑过，含值为 0 的情况）；余额列判断 `result.isAnchor`。
- **锚点值为 0 的余额格**：`isAnchor` 仍 true，可清除（恢复为自动累计值），符合预期。
- **禁用列（`enabled: false`）**：单元格仍可右键（沿用现有行为），「清除该值」照常生效（改的是 `entries`，与 `enabled` 无关）。
- **拖拽 / 重命名 / 启停**：均为列头或独立交互，无冲突。

## 测试（Vitest，`tests/components/MonthlyTable.spec.ts` 新增）

- 动态列：编辑某格 → 右键 → 点「清除该值」→ 该月 `entries` 删除、`isEdited` 恢复 false。
- 余额列：设锚点某格 → 右键 → 点「清除该值」→ 锚点删除、`isAnchor` 恢复 false。
- 禁用：未编辑格右键时，「清除该值」项 `disabled`。
- 顺序：菜单项按「同步 / 清除该值 / 清除下方」顺序出现（余额列无「同步」项）。

## 不做的事

- 不做「清除整列」「清除上方」等其他右键项。
- 不做二次确认弹窗。
- 不改 `ContextMenu.vue`、`useStore.ts`、`types.ts`。
