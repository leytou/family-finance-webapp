# 动态列明细模式设计

- 日期：2026-06-21
- 状态：待评审
- 主题：让可拖拽的动态列也能像"专项"列那样，在某月记录多笔、查看明细、显示合计；默认保持当前单值形态，按需开启。

## 背景

月度流水表现有两类列：

- **专项列**（固定列"专项"）：每月可记录多笔一次性收支（`MilestoneEvent`），单元格显示净合计，悬停看明细，点开增删改每笔。
- **动态列**（`FlowColumn`，可拖拽重排，如"工资""奖金""其他支出"）：每月只能填单个数值，支持"填一次往后延续""标记年度项""右键同步/清除"。

用户希望：动态列也能像专项列那样，在某月记录多笔、查看明细、显示合计；但**默认保持当前单值形态**，按需开启。

## 目标

- 动态列可整体切换为"明细列"，获得与专项列一致的多笔明细能力（多笔、明细、合计）。
- 默认所有动态列为单值，现有行为与数据零变化。
- 切换可逆、零数据损失。
- 明细列保留"填一次往后延续""年度项""右键同步/清除"等现有习惯。
- 复用专项列成熟的编辑/明细交互。

## 现状对比

| 维度 | 专项列 | 动态列（现状） |
|---|---|---|
| 存储 | `plan.events: MilestoneEvent[]`（全局事件列表，按 month 聚合） | `column.entries: Record<YYYYMM, number>`（每月一个数） |
| 每月可记笔数 | 多笔 | 一笔（单值） |
| 合计 | 净合计（正负代数和） | 即该单值 |
| 明细查看 | 悬停弹窗（只读） | 无 |
| 编辑 | `EventEditor`（增删改每笔） | 单元格内联输入单值 |
| 延续 | 脉冲，不延续 | 填一次往后延续（跳过年度项月） |
| 收支归集 | 正→收入，负→支出 | 正→收入，负→支出 |

## 关键设计决策（已与用户确认）

1. **开启层级：整列切换**。一列整体在"单值/明细"间切换，非逐月。
2. **延续：保留**。明细列沿用"填一次往后延续"，延续对象是整组明细。
3. **底层数据：统一升级**。每列"每月一组明细"，单值=一组一笔；切换只改 UI 模式，零损失可逆。
4. **切换入口：单元格右键菜单**。复用现有 `ContextMenu`，加"切换为明细列/切回单值"项。
5. **交互：照搬专项**。格子显示合计、悬停看明细（只读）、点开用编辑器增删改。

## 数据模型（统一升级）

新增单笔明细类型：

```ts
export interface ColumnItem {
  id: string
  name: string    // 可空；单值列那笔 name=''。明细列可填，空则在 UI 以列名占位
  amount: number  // 元；正=收入，负=支出
}
```

`FlowColumn` 升级：

```ts
export interface FlowColumn {
  id: string
  name: string
  itemSets: Record<number, ColumnItem[]>  // 替代 entries；key=YYYYMM，value=该月手填明细组
  yearlyMonths?: Record<number, true>      // 不变；标记月整组不参与往后延续
  enabled?: boolean                        // 不变
  mode?: 'single' | 'detail'               // 新增；缺省=单值
}
```

要点：

- `itemSets[month]` 缺失 = 该月未手填 → 走延续；存在（哪怕空数组）= 手填过。
- 单值模式：每组恰好一笔（`name=''`），`amount` 即原单值。
- 明细模式：每组可多笔。
- `mode` 仅控制 UI（是否允许多笔编辑、是否显示明细入口/笔数角标），不参与计算。计算永远基于组的代数合计。
- 不变项：`MilestoneEvent` / `plan.events`（专项列）原样保留，与本特性并行；两者结构相似但不共用存储。

## 延续与年度项规则

延续逻辑从"值"升级为"整组"，规则与现状一致：

- 解析某月某列：若 `itemSets` 有该月 → 用该月组（手填）；否则向前找最近的、未被 `yearlyMonths` 标记的手填月 → 沿用其整组；都没有 → 空（合计 0）。
- `yearlyMonths` 标记的月：其整组只在该月生效，不作为后续月的延续来源（与单值列年度项语义一致）。
- 合计 = 沿用组的代数和；`isEdited` = 该月是否在 `itemSets` 中（含空组）。

## 切换与数据迁移

**切换（mode）**：单元格右键菜单项。切换单纯改 `column.mode`，不动 `itemSets`，零损失。

- 单值 → 明细：原每组那笔（`name=''`）保留，UI 开放多笔编辑。
- 明细 → 单值：`itemSets` 原样保留，UI 收起为只显示合计；切回明细笔仍在。

**数据迁移（旧 `entries` → `itemSets`）**：虽项目约定"不需兼容旧数据"，但为避免用户现有计划数据丢失，建议提供一次性轻量迁移——load 时若发现旧 `entries` 字段，转为 `itemSets`（每个值 → 一笔 `{ id, name: '', amount }`），`mode` 默认单值，`version` bump。迁移后行为与现状完全等价。

> 已确认：采用此自动迁移，用户现有数据无感保留。

## UI 交互

- **单元格显示**：合计（代数和），紧凑不变，与现在一致（**不显示笔数角标**）。是否多笔、具体每笔，靠悬停或点开查看。
- **悬停**（明细列，组笔数 > 0）：只读明细弹窗，列每笔 `name + amount + 合计`。复用 `EventDetailPopover` 形态。
- **点开编辑**（明细列）：弹出明细组编辑器，每行 `name + amount`，可增删，底部"完成"。复用 `EventEditor` 形态（建议抽取通用组件或参数化复用）。**打开时默认 3 行**（已有项保留，不足 3 行补空行）；需要更多再点"添加"。合计实时显示。改的是该月组；因延续整组，后续延续月自动跟随。
- **单值列**：维持现有单元格内联单值编辑（等价于一笔 `name=''` 的快捷编辑），不弹编辑器。
- **右键菜单**（`ContextMenu`）：现有"同步到下方 / 清除该值 / 清除下方编辑值"保留，操作对象改为整组；新增"切换为明细列 / 切回单值"（依 `mode` 显示文案）。
- **拖拽重排、禁用**：明细列与单值列一致，不受影响。

## 影响范围

- **月度表（`MonthlyTable`）**：主战场——单元格渲染、编辑、明细弹窗、右键切换、笔数角标。
- **`useCalculation`**：`resolveColumnValue` / `hasColumnValue` 按 `itemSets` 组重写（返回合计）；`columnValues` 注入的 `amount = 组合计`。收入/支出分流逻辑不变（仍按 `amount` 正负）。
- **`useStore`**：`entries` 相关操作改为 `itemSets`（置值、同步、清除、增删列）；新增 `toggleColumnMode`；load 迁移。
- **年度表（`AnnualTable`）**：列汇总 = 全年各月组合计之和；不展开明细。
- **对比视图、图表、关键指标**：只消费合计，无需改动。
- **序列化 / 导入导出 / 快照 / 锚点**：整组明细随 `PlanData` 完整存读。
- **专项列**：存储与计算不动；仅其编辑器（`EventEditor`）默认行数 0 → 3（评审统一）。

## 边界处理

- 明细组中某笔 `name` 空 或 `amount` 非法：该笔忽略不计（与 `EventEditor` 一致）。
- 组内全笔失效/被删空：视为该月未手填（回到延续或 0）。
- 延续来的月：点开编辑器看到沿用组；改后自该月起新立分界。
- 切换零损失；明细列可拖拽、可禁用。
- 性能：月数有限（数年 × 12），每组笔数小，无虑。

## 测试范围

- 数据模型：单值 = 一组一笔；切换 `mode` 零损失、来回可逆。
- 迁移：旧 `entries` → `itemSets` 后金额、延续、年度项、`mode` 全部等价。
- 延续：整组延续；年度项月不延续；多组分界。
- 收支分流：多笔正负混排，组合计正确归入当月收入/支出、`totalFlow`。
- 编辑器：增删改、空/非法笔过滤、合计实时、空组回退。
- 右键：同步/清除对整组的作用；切换项文案随 `mode`。
- 编辑器：打开默认 3 行（不足补空、已有项保留）；明细弹窗只读。
- 年度表：全年合计正确。
- 序列化：整组明细存读一致；快照/对比不受影响。

## 非目标（YAGNI）

- 不做列级"收入列/支出列"类型（仍按正负自动分流）。
- 不做明细列的"按笔延续"（只整组延续）。
- 不做年度表的明细展开。
- 不改动专项列。
- 不做明细笔的备注/日期/分类等扩展字段（仅 `name + amount`）。

## 已确认事项（评审敲定）

1. **旧数据迁移**：采用自动迁移（`entries → itemSets`，每值一笔 `name=''`），用户现有数据无感保留。
2. **笔数角标**：不做。单元格只显示合计，多笔与否靠悬停/点开查看。
3. **编辑器默认 3 行**：明细组编辑器与专项列编辑器（`EventEditor`）打开时均默认 3 行（已有项保留、不足补空），需要再添加。专项列编辑器现状为 0 行（"暂无事件，点添加"），一并统一。
