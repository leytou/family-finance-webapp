# 起始月份选择器交互重设计

> 日期：2026-06-13
> 状态：已确认，待实现
> 取代：`docs/superpowers/specs/2026-06-12-起始月份校验与进位-design.md`（基于自由文本输入框的方案）

## 背景

起始月份输入框（`src/App.vue:45-57`）当前是「自由文本 + blur 规范化 + 红框回退」方案（2026-06-12 选定的方案 B）。该方案在实际运行中暴露出一组问题：

1. **红框根本不显示**：非法态 class 仅有 `border-red-500`，丢掉了 `border` 类，`presetUno` 下 `border-red-500` 只设颜色不设宽度/样式 → 边框消失而非变红（`App.vue:53`）。
2. **非法值残留在输入框**：`:value` 单向绑定下，`setStartMonth` 失败不写值，state 不变，Vue 因 prop 未变而跳过 DOM patch → 用户输入的非法字符串不会被回退（注释「自动回退」的假设有误）。
3. **清空被判非法**：`Number("") === 0`，`normalizeMonth(0) === null` → 弹红框。
4. **科学计数法幽灵接受**：`type="number"` 允许 `2e5`，`Number("2e5") === 200000` 被静默接受并进位。
5. **进位是隐式魔法**：输入 `202613` 会被进位成 `202701`，无任何提示。
6. **反馈不可理解**：只有红框（且不显示），无错误文案。
7. **三种输入框三种交互模式**：起始月份受控、年化收益率实时写、初始存款 v-model，不一致。
8. **代码质量**：模板内 TS 类型断言、`startMonthInvalidTimer` 未在卸载时清理、label 未关联 input（无障碍缺失）。

根因在于：**用自由文本输入框承载一个本质上是「年 + 月（1–12）」的离散值**，于是需要持续修补校验、进位、回退、反馈。本次重设计换一个形态，从结构上消除这些问题。

## 目标

1. 起始月份只能落在合法的 YYYYMM（年份 1000–9999、月份 1–12），**结构上不可能**产生越界、位数不足、非法字符。
2. 交互即点选：年份步进/键入 + 月份格子点选，彻底放弃自由文本输入。
3. 沿用项目既有浮层模式（`useClickOutside`、`fixed z-50`、Esc 关闭、`role` 无障碍），不另起一套。
4. 不改数据模型：`startMonth` 仍为 `number`，与现有所有计算/存储/格式化代码兼容。
5. 在 `month.ts` 补一个中文友好格式化纯函数，可单测。

## 需求澄清结论（2026-06-13）

- **控件形态**：日历面板点选（非单一 YYYYMM 文本框，非年/月分拆下拉）。
- **面板实现**：自定义月份面板（非浏览器原生 `<input type="month">`——后者总带「日」、value 是字符串、各浏览器样式不一）。
- **输入路径**：彻底点选，不保留「直接输入 YYYYMM」的快速路径。
- **数据模型**：`startMonth` 保持 `number`，不改。
- **显示文案**：紧凑态显示 `2026年1月` 中文友好格式。
- **年份编辑**：保留 ◀▶ 步进，且点中年份可键入 4 位数字（校验 4 位整数，非法回退）。这是整条链路里唯一残留的微型自由文本，无月份越界/进位问题。
- **键盘方向键导航**：作为二阶段增强，不进首版。
- **范围**：仅起始月份。不统一年化收益率、初始存款两个输入框（如需，另起 spec）。

## 选定设计

### 1. 组件结构

新增独立组件 `src/components/MonthPicker.vue`，与 `ContextMenu`/`FormulaPopover` 同级。单一职责、可单测、未来可复用。

`App.vue` 的起始月份区域从「input + 3 个 ref + 1 个处理函数 + 模板类型断言」收敛为：

```vue
<div class="flex items-center gap-2">
  <label for="start-month" class="text-xs whitespace-nowrap">起始月份</label>
  <MonthPicker v-model="startMonth" input-id="start-month" />
</div>
```

删除 `App.vue:18-33` 的 `startMonthInvalid`、`startMonthInvalidTimer`、`onStartMonthBlur`，以及模板 `App.vue:48-56` 的 `:value`/`@blur`/`@keydown.enter`/红框 class。

### 2. 数据流

```
MonthPicker (defineModel<number>)        App.vue (computed 桥接)       useStore
┌────────────────────────┐  v-model   ┌─────────────────────┐ set()  ┌──────────────┐
│ 选中年月 → year*100+month│ ────────▶ │ get: startMonth     │ ─────▶ │ setStartMonth│
│                         │ ◀──────── │ set: setStartMonth()│ ◀───── │ (原样写入)   │
└────────────────────────┘            └─────────────────────┘        └──────────────┘
```

- `MonthPicker` 通过 `defineModel<number>()` 双向绑定一个 YYYYMM 数字，组件本身不感知 store。
- App.vue 用 computed 把它桥接到 `setStartMonth`，保留「修改 startMonth 走 store 动作」这一既有约定：

```ts
const { data, setStartMonth } = useStore()
const startMonth = computed({
  get: () => data.value.systemParams.startMonth,
  set: (v: number) => { setStartMonth(v) },
})
```

- 选出来的值恒为合法 YYYYMM（4 位年 × 1–12 月），`setStartMonth` 内的 `normalizeMonth` 对其是恒等 → 原样写入。
- `setStartMonth` / `normalizeMonth` / `isValidYyyyMm` **保留不删**（仍是 `month.ts` 通用能力，且有测试覆盖），只是 MonthPicker 这条路径不再依赖其校验。

### 3. MonthPicker 组件设计

**Props / Model**

```ts
const model = defineModel<number>()                      // YYYYMM，双向；首版仅用 model，不额外 emit 事件
const props = defineProps<{ inputId?: string }>()        // 供 label 关联
```

**内部状态**

- `open: Ref<boolean>` —— 面板是否展开。
- `panelYear: Ref<number>` —— 面板当前展示的年份（独立于已选值，允许翻看其它年而不立即提交），展开时用当前 model 的年份初始化。
- `yearEditing: Ref<boolean>` / `yearInput: Ref<string>` —— 年份键入态。
- `panelRef: Ref<HTMLElement | null>`、`triggerRef` —— 浮层与触发器引用。
- `panelPos: { left, top }` —— 展开时由触发器 `getBoundingClientRect()` 计算的定位坐标。

**紧凑态（触发器）**

```
┌────────────────┐
│ 2026年1月   ▾  │   <button aria-haspopup="dialog" :aria-expanded="open">
└────────────────┘
```

文案由 `formatMonthZh(model)` 生成。点击切换 `open`。

**展开态（浮层）**

```
┌─────────────────────┐
│ ◀    2026年    ▶  │   ◀▶：panelYear -- / ++
│                     │   点中年份 → 切到 4 位输入框（yearEditing）
│  1月  2月  3月      │
│  4月  5月  6月      │   月份格 4 行 × 3 列（1-3 / 4-6 / 7-9 / 10-12），恒为 1–12
│  7月  8月  9月      │   当前已选月 aria-current / 高亮
│ 10月 11月 12月      │   点击 → 提交 year*100+month 并关闭
└─────────────────────┘
```

浮层沿用 `ContextMenu.vue` 模式：`fixed z-50`、`:style="{ left, top }"`、`tabindex="-1"`、`@keyup.escape` 关闭、`onMounted` 聚焦面板容器。点外部关闭用 `useClickOutside(panelRef, () => close())`。

**年份键入校验（唯一残留自由文本，极简）**

```ts
function commitYearInput() {
  const y = Number(yearInput.value)
  if (Number.isInteger(y) && y >= 1000 && y <= 9999) {
    panelYear.value = y
  }
  // 非法（空、非整数、越出 1000–9999）→ 不改 panelYear，自动回退
  yearEditing.value = false
}
```

年份没有「月份越界」概念，任意 4 位年都合法；不存在进位、不存在 `Number("")` 副作用（空值直接回退）。触发：blur 或 Enter。

**关闭策略**

- 点外部（`useClickOutside`）。
- Esc（`@keyup.escape`）。
- 选定某月（提交并关闭）。

### 4. 显示格式化（src/utils/month.ts）

新增纯函数，紧凑态文案与面板头部共用：

```ts
/** YYYYMM → 「2026年1月」中文友好格式 */
export function formatMonthZh(yyyymm: number): string {
  const year = Math.floor(yyyymm / 100)
  const month = yyyymm % 100
  return `${year}年${month}月`
}
```

现有 `formatMonth`（`YYYY-MM`）不动——快照名等仍在用（`useStore.ts:320`）。

### 5. 无障碍

- 触发器：`aria-haspopup="dialog"`、`:aria-expanded="open"`、`:aria-controls`、`id`（供 `<label for>` 关联）。
- 面板：`role="dialog"`、`aria-label="选择起始月份"`、`tabindex="-1"`、挂载后聚焦。
- 月份按钮：已选月 `aria-current="true"`；`aria-label` 形如 `2026年1月`。
- 键板：Tab 进入、Esc 关闭（首版）；方向键在 12 格间移动 + Enter 选定属二阶段增强。

### 7. 非目标（YAGNI）

- 不改数据模型（`startMonth` 仍 `number`）。
- 不统一年化收益率、初始存款两个输入框。
- 不设年份业务上下限（保持 1000–9999，与 `isValidYyyyMm` 一致）。
- 不做「禁用某些月份」逻辑。
- 不做月份方向键导航（二阶段增强）。
- 不删除 `setStartMonth` / `normalizeMonth` / `isValidYyyyMm`（保留为通用能力）。

## 对原方案问题的覆盖核对

| 原问题 | 本设计如何消除 |
|---|---|
| 红框不显示 / 非法值残留 / 清空判非法 / 科学计数法 | 不再有起始月份自由文本输入，无 `Number()`、无 `:value` 回退、无红框 |
| 进位隐式魔法 | 月份只能 1–12，结构上不可能越界，无需进位 |
| 反馈不可理解 | 选中即所见，所见即所存；年份键入非法仅在面板内回退，不影响主数据 |
| 三种输入框模式不一致 | 本次仅起始月份；其余两个输入框不在范围（见非目标） |
| 模板类型断言 / timer 未清理 / label 未关联 | 组件内置 `label for` 关联；面板内自管理状态，随组件卸载而清理 |
