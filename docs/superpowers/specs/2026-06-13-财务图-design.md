# 财务趋势图（组合图：累计储蓄折线 + 收支流柱状）

## 目标

新增一个全屏「财务趋势图」视图，把当前激活方案的 60 个月投射数据可视化为**组合图**：累计储蓄折线（存量）+ 收入/支支柱状（流量）。支持「按月 / 按年」粒度切换。让用户从「逐行看表格数字」升级到「一眼看趋势与收支结构」——核心回答「我的钱会怎么涨、每月赚的够不够花的」。

典型场景：调完参数（年化收益、某项支出）后切到图表，直观看到累计储蓄曲线的变化与每月净流的正负。

## 背景

- 现有视图全是表格：`AnnualTable`（年度汇总，占 35% 高度）、`MonthlyTable`（月度明细），外加 `ComparisonView`（多方案/快照偏差对比）。**没有任何数据可视化图表**。
- 数据源已齐全：`App.vue` 有 `results = computed(() => calculate(data.value))`，`MonthResult` 含 `monthlyIncome` / `monthlyExpense` / `monthlyBalance` / `investReturn` / `cumSavings`，投射 60 个月（`PROJECTION_MONTHS`）。
- 年度聚合口径：`AnnualTable` 按**自然年**分组（`Math.floor(month / 100)`），年末存款取该年最后一月 `cumSavings`。图表的「按年」**必须沿用同一口径**，避免图表与年度表数字打架。
- 项目目前无任何图表库依赖；样式用 UnoCSS，金额/月份格式化在 `utils/format.ts`、`utils/month.ts`。

## 架构选择

- **技术方案 A（采用）**：**ECharts 按需引入**。组合图（柱+折线）、双 Y 轴、月年聚合切换、tooltip、图例全部开箱即用，中文文档完善，能最快做出高质量交互图。代价是体积（按需引入约 300–400KB），对本地规划工具可接受。
- 方案 B（否决）：**Chart.js**。轻量（约 200KB），组合图可用，但「支出镜像负值柱」「月年聚合切换」「tooltip 中文格式化」需较多手工配置，定制度不及 ECharts。
- 方案 C（否决）：**手写 SVG**。零依赖、体积最小、与项目轻量风格一致，但坐标轴、双轴刻度对齐、tooltip、图例、响应式、动画全要自实现，工作量大、后续维护成本高。

视图集成采用「独立全屏视图」（与现有「对比」视图平级），不挤压表格空间，信息密集的组合图也能获得足够展示面积。

## 范围

- 新组件 `src/components/FinanceChart.vue`，props 接收 `results: MonthResult[]`（与 `MonthlyTable` / `AnnualTable` 完全一致的接入方式）。
- `App.vue` 视图状态：把现有 `showComparison`（布尔）升级为单一状态 `activeView: 'table' | 'chart' | 'comparison'`，三视图互斥；header 在「对比」旁新增「图表」按钮。
- 组件内「按月 / 按年」粒度切换。
- 新增年度聚合纯函数 `aggregateByYear(results)`（与 `buildComparison` 并列，放 `useCalculation.ts`）。
- 引入 `echarts` 并按需 register。
- **不做**（见非目标）：多方案折线叠加、快照预计线叠加、数据缩放 dataZoom。

## 数据流

```
header 点「图表」按钮
  → activeView = 'chart'
  → main 渲染 <FinanceChart :results="results" />（复用现有 results computed，零计算改动）

FinanceChart 内部：
  props.results + granularity('month' | 'year')
    → computed chartData：
        按月 → 直接映射 monthlyIncome / -monthlyExpense / cumSavings
        按年 → aggregateByYear(results) 聚合后再映射
    → watch(chartData) → chart.setOption(option)
  切粒度 → granularity 变 → chartData 重算 → setOption 增量更新
  容器 resize → ResizeObserver → chart.resize()
  组件卸载 → chart.dispose()
```

### 3. `FinanceChart.vue`

```ts
const props = defineProps<{ results: MonthResult[] }>()

const granularity = ref<'month' | 'year'>('month')

// 月视图标签：YY/MM 紧凑格式（如 26/01）；年视图：年份（2026）
function formatAxisLabel(month: number, isMonth: boolean): string { ... }

// 把 results + granularity 转成 { categories, income, expense, cumSavings }
const chartData = computed(() => {
  if (granularity.value === 'year') {
    const years = aggregateByYear(props.results)
    return {
      categories: years.map(p => String(p.year)),
      income: years.map(p => p.income),
      expense: years.map(p => -p.expense),      // 取负，向下画
      cumSavings: years.map(p => p.cumSavings),
    }
  }
  return {
    categories: props.results.map(r => formatAxisLabel(r.month, true)),
    income: props.results.map(r => r.monthlyIncome),
    expense: props.results.map(r => -r.monthlyExpense),   // 取负
    cumSavings: props.results.map(r => r.cumSavings),
  }
})
```

option 构建抽成纯函数 `buildChartOption(chartData)` 便于单测（不依赖实例）：

- **双 yAxis**：`yAxis[0]` 流量（左轴，可正可负，`alignTicks: true`）、`yAxis[1]` 累计储蓄（右轴）。
- **三个 series**：
  - `bar` 收入（`yAxisIndex: 0`，正值）
  - `bar` 支出（`yAxisIndex: 0`，负值）
  - `line` 累计储蓄（`yAxisIndex: 1`，`smooth` 视效果、`symbol: 'none'` 或小圆点）
- **tooltip**：`trigger: 'axis'`，formatter 用 `formatCurrency` 中文标注「收入 / 支出 / 累计储蓄」。
- **legend**：顶部三项，可点击隐藏系列。
- **grid**：预留左右轴与底部标签空间。

生命周期与响应式：

```ts
let chart: echarts.ECharts | null = null
const el = ref<HTMLElement>()

onMounted(() => {
  chart = echarts.init(el.value!)
  chart.setOption(buildChartOption(chartData.value))
  // ResizeObserver 监听容器尺寸 → chart.resize()
})

watch(chartData, () => chart?.setOption(buildChartOption(chartData.value)))

onUnmounted(() => {
  // 断开 ResizeObserver
  chart?.dispose()
  chart = null
})
```

模板：一个撑满父容器的 `<div ref="el" class="h-full w-full" />`，外加右上角的「按月 / 按年」切换控件（按钮或 segmented control）。

### 4. `App.vue` 视图状态升级

- `showComparison: Ref<boolean>` → `activeView: Ref<'table' | 'chart' | 'comparison'>`（默认 `'table'`）。
- header 「对比」按钮旁新增「图表」按钮，二者互斥高亮：
  - 点「图表」→ `activeView = 'chart'`；再点（已激活）→ 回 `'table'`。
  - 点「对比」→ `activeView = 'comparison'`（原 `showComparison` 逻辑迁移）。
- main 区条件渲染：
  - `activeView === 'comparison'` → `<ComparisonView @close="activeView = 'table'" />`
  - `activeView === 'chart'` → `<FinanceChart :results="results" />`（全屏 `flex-1`）
  - 否则 → 原 年度表 + 月度表 布局

## 配色与标签

- **配色**：收入 `#16a34a`（green-600）、支出 `#dc2626`（red-600）、累计储蓄 `#2563eb`（blue-600），与应用现有绿色基调协调。
- **轴标签**：月视图 `YY/MM`（`26/01`），标签过密时依赖 ECharts 自动间隔显示；年视图直接用年份（`2026`）。
- **金额**：tooltip 内一律 `formatCurrency`（整数千分位）。

## 边界与错误处理

- **累计储蓄为负**：折线下探到 0 轴以下，Y 轴自适应，正常显示。
- **全 0 数据**：收入/支出柱不显示、累计储蓄平线，图表不报错。
- **恒定 60 月**：`calculate` 恒返回 60 个月，无需处理空数组；若未来 `results` 为空，`aggregateByYear` 返回 `[]`，图表显示空白（可后续加空状态）。
- **年度首尾不完整**：投射跨自然年时，首年/末年可能不满 12 月，按实际包含的月份聚合（与 `AnnualTable` 行为一致）。
- **组件卸载**：`dispose` + 断开 `ResizeObserver`，避免内存泄漏与重复 resize。
- **数据未变**：`watch(chartData)` 仅在数据或粒度变化时 `setOption`，避免无谓重绘。

## 测试方案（Vitest）

### `tests/composables/useCalculation.spec.ts` — 新增 `aggregateByYear` 组

- 单年：收入/支出求和正确、年末累计 = 该年最后一月 `cumSavings`。
- 跨年边界：同一年份月份归到同组、跨年正确切分。
- 首年/末年不满 12 月：按实际月份聚合。
- 与 `AnnualTable` 口径一致：对同一 `results`，`aggregateByYear` 的 `cumSavings` 等于 `AnnualTable` 的年末存款（回归保护）。

### `tests/components/FinanceChart.spec.ts` — 转换与粒度

- `buildChartOption` 纯函数：支出系列取负、系列顺序（收入/支出/累计）、双轴 `yAxisIndex` 正确。
- 月视图 `categories` 用 `YY/MM`、年视图用年份。
- 粒度切换：`granularity` 由 `month` → `year` 时，`chartData` 由 60 点变为年数点。（组件层可用 `@vue/test-utils` 挂载并 mock `echarts`，断言 `setOption` 被以正确 option 调用；或仅测 `buildChartOption` / `chartData` 计算属性，不直接测实例。）

## 非目标（YAGNI）

- **多方案折线叠加**：本期只画当前激活方案；多方案趋势对比作为后续扩展点。
- **快照预计线叠加**：实际 vs 预计偏差已由现有「对比」视图覆盖，图表不重复。
- **数据缩放 dataZoom**：60 个月 / 5 年数据量不大，不做区间缩放。
- **图表导出图片**：用户可截图，不内建导出。
- **可编辑图表**：图表只读，编辑仍走表格。
- **手写图表主题切换**：沿用固定配色，不做深色模式适配。
