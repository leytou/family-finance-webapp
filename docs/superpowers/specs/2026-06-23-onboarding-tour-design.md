# 教程引导（Onboarding Tour）设计

**日期**：2026-06-23
**状态**：待审查

## 背景

应用功能较多，核心流程（定参数 → 填收支 → 看结果）和若干高级功能（公积金专区、多方案对比、储蓄目标核对/锚点）对新用户不够直观。需要一套教程引导，降低上手门槛，并让老用户能随时回查高级功能。

## 目标

1. **带新用户跑通首次**：从打开应用到看到第一份可看的规划结果。
2. **讲清不直观的高级功能**：公积金、多方案对比、锚点。
3. **可随时重看、不打扰老用户**：首次自动播一次（可跳过），之后只在用户主动点「教程」时出现。

## 非目标（YAGNI）

- 不做交互式操作引导（不监听用户是否真的填写，纯讲解）。
- 不做多语言；文案固定中文。
- 不在引导里嵌视频/动图。
- 不做「未完成教程提醒」。
- 计算器、专项事件、动态列进阶等暂不单独成主题，后续可加。

## 整体设计

### 两种触发方式

1. **首次自动播放「快速入门」**
   打开应用时，若本地未标记「已看过」，自动播放「快速入门」。
   看完或点「跳过」后写入「已看过」标记，以后打开不再自动弹。
   **含老用户升级后首次**（只要标记不存在就播一次），但可一键跳过。

2. **顶部「教程」入口（常驻）**
   顶部导航增加「教程」按钮，点开弹出**分主题菜单**，选某主题即播放对应 Tour。与「更多」按钮并列。

   菜单项：
   | 主题 | 步数 | 内容大纲 |
   |---|---|---|
   | 🚀 快速入门 | 5 | 定参数 → 填收支 → 看结果 → 看趋势 |
   | 🏠 公积金专区 | ~3 | 开启开关 → 缴存 / 月冲 / 提取 → 结息月份 |
   | 📊 多方案对比 | ~2 | 方案标签「+」建/复制方案 → 切「对比」视图并排比较 |
   | 🎯 储蓄目标核对（锚点） | ~2 | 月度表「累计结余」列点某月设预期额 → 看是否达标/偏差 |
   | 🔁 重看全部 | — | 把以上主题步骤串成一个连续序列一次播完 |

> 「重看全部」= 串联所有主题步骤连续播放。**任何手动播放（含重看全部）都不改变「已看过」标记**，该标记仅控制「首次是否自动播」。

### 呈现形态：聚光灯式 Tour

- 半透明遮罩盖住全屏，**只照亮正在讲的那个元素**，其余变暗。
- 紧贴高亮元素的**气泡**：标题 + 一句说明 + 「上一步 / 下一步 / 跳过」。
- **纯讲解**：用户只看 + 点「下一步」，不要求动手操作。
- 点遮罩空白处或「跳过」可立即结束。

> 用户视角：像有一个聚光灯跟着讲解走，一格格点「下一步」，随时能走开。

### 「快速入门」5 步内容

| 步 | 指向区域 | 讲解要点 |
|---|---|---|
| 1 | 参数行 · 起止月份 | 先定规划期限：从哪月到哪月 |
| 2 | 参数行 · 收益率 / 初始存款 | 设年化收益率（投资回报）和手头初始存款 |
| 3 | 月度流水表（核心） | 在这里添加收入/支出项、逐月填金额——数据都从这来 |
| 4 | 关键指标条 / 年度汇总 | 填的数字实时反映在这里：指标看整体、年度表看每年 |
| 5 | 顶部「图表」按钮 | 点「图表」切换到趋势图看储蓄增长 |

## 技术选型

**采用 driver.js（v1.x）**。

- 轻量（压缩后约 5KB）、主流活跃维护、框架无关（Vue 3 直接用，无需 wrapper）。
- 配置式 `steps`（element 选择器 + popover 标题/说明）、程序化驱动（`drive()`/`move()`/`destroy()`）、`onDestroyed` 回调（用于写「已看过」标记）、内置滚动到元素与 popover 自动重定位。
- 相比 shepherd.js 更轻；相比 vue-tour（Vue 2 为主、维护不活跃）更适配 Vue 3。

**需做的适配**：
- 引入 `driver.js` 与其 `driver.css`。
- 自定义 popover 样式，贴合项目配色 token（brand 蓝、ink 正文、surface 背景）与字体（font-mono），**不用 driver.js 默认蓝**。
- 为 Tour 要指向的元素加稳定定位属性 `data-tour="<id>"`（driver.js 靠选择器定位，现有 `data-testid` 可复用或新增 `data-tour`）。

## 文件结构（建议）

- `src/composables/useTour.ts` —— Tour 编排：各主题 `steps` 数据、播放控制（`play(topic)`）、「已看过」标记读写。
- `src/data/tours.ts`（可选）—— 各主题步骤文案集中存放，便于维护。
- `src/components/TourMenu.vue` —— 顶部「教程」按钮 + 主题下拉菜单。

## 集成点

1. **`App.vue` 顶部导航**：在「更多」按钮旁加 `<TourMenu />`。
2. **`App.vue` `onMounted`**：读取「已看过」标记，未看过则调用 `useTour().play('quickstart')` 自动播放。
3. **目标元素**：为下列区域元素加 `data-tour` 属性——参数行各字段、月度流水表区、关键指标条、年度汇总区、「图表」按钮、方案标签「+」、「对比」按钮、累计结余列、公积金开关等。

## 「已看过」持久化

- localStorage key：`family-finance-tour-seen`（**独立于**数据 key `family-finance-plan`，避免与方案数据耦合）。
- 值：boolean。首次自动播放「快速入门」看毕或跳过即置 `true`；此后不再自动播。
- 其他主题的手动播放不影响该标记（只控「首次自动播不播」）。

## 边界处理

- **目标元素不在屏内**：driver.js 内置 `scrollIntoView`，自动滚到可见再高亮。
- **公积金主题依赖开关**：公积金的缴存/月冲/提取 UI 仅在 `fundEnabled` 时存在。该主题首步处理开关：已开启则跳过该步、直接讲配置；未开启则首步引导用户开启，再讲缴存/月冲/提取/结息。
- **对比主题需切视图**：对比 Tour 步骤涉及「对比」视图，播放时先切到对比视图再高亮。
- **窗口缩放 / 滚动**：driver.js 内置 popover 重定位。
- **Tour 进行中用户手动切视图/操作**：当前 Tour 仍指向原元素；若元素被卸载则步骤提前结束（driver.js 行为），可接受。

## 样式

- popover 背景用 `bg-surface`、正文 `text-ink`、主按钮「下一步」用 `brand-600`，与现有按钮一致。
- 字体沿用 `font-mono` / 现有尺寸体系，不引入新字体。
- 遮罩色用中性半透明黑，高亮框边沿用 `brand-600`。

## 测试策略（Vitest）

1. **`useTour` 单元测试**：
   - 各主题 `steps` 数据结构正确（每步有 element 选择器 + popover 文案）。
   - 「已看过」标记读写：未标记返回 false、标记后返回 true、独立 key 不污染方案数据。
2. **播放集成测试**：挂载 App，mock localStorage 未标记 → 断言首次自动调用 `play('quickstart')`；标记后不再自动播。
3. **菜单交互**：点「教程」弹出菜单、点某主题调用对应 `play`。
4. driver.js 本身为外部依赖，不测其内部，只测我们的编排与集成。

## 风险与取舍

- **driver.js 元素定位依赖 DOM 存在**：公积金/对比等主题元素受开关/视图影响，需在编排里保证前置条件（见边界处理）。
- **首次自动播对老用户的打扰**：通过「可一键跳过 + 跳过即标记」缓解；选择「没看过都播一次」是为了让所有人知道有教程存在。
- **样式适配工作量**：driver.js 默认样式需覆盖以贴合项目，已计入实现工作量。
