# 多方案对比 — 设计文档

## 1. 背景与目标

现有应用只能维护一份财务规划（单个 `PlanData`）。当用户想评估买房、育儿等人生大事件时，无法把「做这件事」和「不做」、或同一目标的不同方案放在一起对比。

本设计引入**多方案**能力：用户可创建多份完整独立的规划（如「买房方案」「租房方案」），通过顶部 TAB 切换查看，并在专门的对比页并排比较关键指标。

买房、育儿这类大事件本身仍用现有的「现金流列」表达，无需新增模型——对比能力由「多套方案」这一层提供。

## 2. 数据结构

在现有 `PlanData` 外包一层 `Workspace`：

```ts
interface Scenario {
  id: string
  name: string        // 如「买房方案」「租房方案」
  plan: PlanData      // 完整独立副本，含各自的现金流列、锚点、系统参数
}

interface Workspace {
  version: number
  scenarios: Scenario[]
  activeId: string    // 当前激活（查看/编辑）的方案 id
}
```

要点：
- 每个方案是**完整独立**的 `PlanData`，连 `startMonth`、`annualRate` 也各存各的，方案间互不影响。
- localStorage 存的是整个 `Workspace`（沿用现有的 300ms 防抖保存）。
- `PlanData` 结构本身不变。

### 数据迁移

加载时若发现 localStorage 中是旧的单 `PlanData` 格式（无 `scenarios` 字段），自动包装成一个名为「默认方案」的 `Scenario`，置为 `activeId`。这样用户已有数据不丢失。迁移为一次性，写回后即为 `Workspace` 格式。

## 3. 状态管理（useStore 改造）

`useStore` 内部持有 `Workspace`，但对外暴露的接口尽量保持不变，使 `App.vue` / `MonthlyTable` / `AnnualTable` 几乎无需改动：

- `data`：改为指向「当前激活方案的 `plan`」（computed / 代理），现有的 `data.value.columns`、`data.value.systemParams` 等访问方式不变。
- 现有列/锚点操作（`addColumn`、`removeColumn`、`renameColumn`、`updateColumnEntry`、`addAnchor`、`removeAnchor`）：作用于当前激活方案，签名不变。
- `reset`：仅重置**当前方案**的 plan 为默认空白 plan，不影响其他方案。

新增方案级操作：
- `addScenario()`：创建空白方案，立即激活，返回新方案（供 UI 进入命名编辑态）。
- `duplicateScenario()`：深拷贝当前方案的 plan，作为新方案，立即激活并返回。
- `removeScenario(id)`：删除指定方案；若仅剩一个方案则禁止删除。
- `renameScenario(id, name)`：重命名。
- `switchScenario(id)`：切换 `activeId`。

移除：`exportData` 及相关导出功能。

## 4. 顶部方案 TAB（App.vue header）

header 标题右侧用**横向 TAB** 展示所有方案，其后跟两个操作按钮；最右侧保留起始月份、收益率、重置，并新增对比入口：

```
家庭财务规划 │ [买房方案] [租房方案✕] […] │ [新建方案] [复制当前]   起始月份 收益率 重置 [对比]
```

交互：
- **切换**：点 tab 切换激活方案，当前方案高亮；年度/月度表实时重算为该方案数据，界面其余部分不变。
- **新建方案**：点击后立即创建空白方案并激活，且**自动进入方案名编辑态**（tab 上行内 input 聚焦，可直接打字命名）。
- **复制当前**：点击后深拷贝当前方案为新方案并激活，同样**自动进入命名编辑态**。
- **重命名**：双击 tab 上的方案名进入行内编辑态（回车/失焦确认，Esc 取消），复用现有列重命名交互模式。
- **删除**：hover 到某个 tab 时，名字右侧浮出 `✕` 图标；点击后 confirm 一次再删除。仅剩一个方案时不显示 `✕`（禁止删除）。

`startMonth`、`annualRate` 编辑、`重置` 均只作用于当前激活方案（语义不变，因 `data` 已指向当前方案）。

## 5. 对比页

header 最右侧 **[对比]** 按钮进入对比视图：切换主区域内容（年度表+月度表 ⇄ 对比页），再次点击或关闭按钮返回编辑视图。

对比页内容（信息密度高、数字为主）：

- **方案选择**：顶部勾选要对比的方案（默认全选），至少 2 个。
- **关键指标并排表**：每个方案一列，行为关键指标：
  - 各年末累计储蓄（第 1 年末 … 第 5 年末）
  - 5 年总收入
  - 5 年总支出
  - 5 年净储蓄
  - 最终累计储蓄
  - 期间最低储蓄点（最低的月末累计储蓄，用于发现某方案是否会被大额支出掏空到负）
- **差异高亮**：以勾选的第一个方案为基准，其他方案的数字标注与基准的差额（如 `+12万` 绿色 / `-30万` 红色），快速看出优劣。

实现：复用现有 `calculate()`，对每个选中方案各算一遍 `MonthResult[]`，再聚合出上述指标。不修改核心计算逻辑。

**不做**（YAGNI）：本期不做趋势折线/柱状图，纯并排指标表已足够。

## 6. 不在本期范围

- 共享基础数据 + 差异项的方案模型（本期为完全独立副本）。
- 导出功能（移除）。
- 对比页图表可视化。

## 7. 测试要点

- `Workspace` 数据迁移：旧单 `PlanData` 正确包装为默认方案。
- 方案操作：新建/复制/删除/重命名/切换的状态正确性；删除最后一个方案被禁止；复制为深拷贝（改新方案不影响原方案）。
- `useStore` 对外接口在多方案下行为正确（列/锚点操作只作用于当前方案）。
- 对比页指标聚合：各年末储蓄、总收支、最低储蓄点计算正确；差额以基准方案为参照。
