# 家庭财务规划 · UI 重新设计（浅色金融终端）

- 日期：2026-06-19
- 作者：Leytou + Claude
- 状态：待评审

## 1. 背景与目标

当前应用是一个信息密度极高的「电子表格型财务工作台」，骨架（数据模型、表格交互、中式红绿语义配色）已经相当扎实，但「皮肤」偏原始通用：

- 零自定义字体，全用系统默认无衬线
- 用 emoji（📥🔄🧮👁）和 ASCII 当图标，质感不一致
- 所有按钮都是同款描边方块，没有主次层级
- 白底单调，首屏没有视觉焦点
- 负数用斜体表达，不直觉

本次目标：**在不动任何功能、计算逻辑、数据模型的前提下，把整个界面重新设计为「浅色金融终端」风格**——专业、冷静、信息吞吐量高，同时通过适度增强补上视觉焦点，让首屏更易读、更有高级感。

### 设计目标

1. **保留高信息密度**：表格仍是主体，数字仍是主角，单屏可见数据量不降反升
2. **统一视觉语言**：所有页面、所有弹窗遵循同一套设计 token 和组件规范，消除现有的质感参差
3. **补上视觉焦点**：新增「关键指标条」作为首屏锚点，解决"白底无焦点"
4. **零功能风险**：纯视觉改造 + 用现有计算结果做展示增强，不碰计算与数据

### 非目标（明确不做）

- 不改 `types.ts` 数据模型
- 不改 `useCalculation.ts` 及任何计算逻辑
- 不改 `useStore` / `useHistory` / `useComparison` / `useCalculator` 业务逻辑
- 不新增/删除任何功能、不改变交互行为（单元格编辑、列拖拽、公式浮窗、快捷键等全部保留）
- 不做暗色模式（本次只交付浅色）

## 2. 设计原则

1. **数据优先**：所有装饰服务于「读数字」。等宽字体 + 右对齐 + `tabular-nums` 是底线
2. **终端气质**：等宽字体贯穿、大写小标签 + 字距、状态点、走势线、编号式区块标题
3. **冷调克制**：大面积冷调浅灰白，色彩只用于语义（朱砂/竹青/靛蓝/琥珀），不堆砌
4. **一致性胜过创意**：弹窗、按钮、表格、图标全部走统一规范；创意体现在整体气质，而非各处花样
5. **保留语义资产**：朱砂＝收入/正向、竹青＝支出/负向、靛蓝＝交互主色、琥珀＝告警——这套已有的中式语义配色直接继承，并贯穿到图表与表格

## 3. 设计 Token 系统

在 `uno.config.ts` 现有主题基础上扩展，并新增一组「表面/线条」语义别名，全局通过 CSS 变量统一调度。

### 3.1 配色

保留现有语义色（已在 `uno.config.ts` 定义，值不变）：

| Token | 含义 | 主值 |
|---|---|---|
| `brand` (indigo) | 交互主色 / 选中 / 当前 | `#4f46e5` |
| `positive` (zhusha) | 收入 / 正向差额 | `#c0504d` |
| `negative` (zhuqing) | 支出 / 负向差额 | `#6b8e7b` |
| `warning` (amber) | 告警 / 截断提示 | `#d97706` |
| `success` (green) | 操作成功 | 标准绿 |
| `danger` (red) | 操作失败 / 删除 | 标准红 |
| `neutral` (slate) | 中性文字/边框/背景 | slate 阶 |

新增「表面层」语义别名（基于 slate 派生的冷调）：

| Token | 含义 | 值 |
|---|---|---|
| `canvas` | 应用底色（冷调浅灰，非纯白） | `#f3f5fa` |
| `surface` | 面板/卡片/表格底 | `#ffffff` |
| `surface-2` | 次级表面（表头、参数行底） | `#f8fafc` |
| `line` | 主分隔线/边框 | `#e4e8f1` |
| `line-soft` | 次级分隔线（行分隔） | `#eef1f7` |

文字层次（基于 slate）：

| Token | 用途 | 值 |
|---|---|---|
| `text` | 主文字 | `#1a2233` |
| `text-2` | 次级文字（列名、说明） | `#5b6678` |
| `text-3` | 弱文字（表头、占位） | `#8a93a6` |

> 实现方式：在 `uno.config.ts` 的 `theme.colors` 下新增 `canvas / surface / line / line-soft / ink（text 系列）` 等键，值如上；组件中可直接用 `bg-canvas`、`text-ink-2`、`border-line` 等原子类。

### 3.2 字体

| 用途 | 字体栈 | 说明 |
|---|---|---|
| 数字 / 标签 / 英文标题 / 终端感文本 | `'JetBrains Mono','Cascadia Code','Consolas',ui-monospace,monospace` | 贯穿全局，终端灵魂 |
| 中文正文 / 中文标题 | `'PingFang SC','Microsoft YaHei',system-ui,sans-serif` | 系统黑体，不内嵌，避免打包膨胀 |

- **等宽字体引入**：优先 JetBrains Mono（OFL 开源）。鉴于项目用 `vite-plugin-singlefile` 打包成单 HTML，webfont 内嵌会增大体积，**实现时优先用系统等宽降级（Cascadia Code/Consolas），JetBrains Mono 作为 CDN `@import` 渐进增强**；若体积可接受（<200KB 子集）再考虑内嵌 woff2。决策留到实现计划阶段。
- 所有数字统一 `font-variant-numeric: tabular-nums`，右对齐。
- 中文不强行等宽，避免排版僵硬。

### 3.3 字号体系

| 角色 | 字号 | 用途 |
|---|---|---|
| hero 指标数字 | 22–24px | 关键指标条 |
| 区块标题 / 大标题 | 14–16px | 页面、section |
| 正文 | 13px | 说明、对比表 |
| 表格数字 / 参数 | 11–11.5px | 月度表、年度表、参数行（保持现有密度） |
| 标签 / 表头 / 注释 | 9.5–10.5px | mono 大写带字距 |

> 表格区维持现状的 `text-[11px]` 高密度，不放大；放大发生在「留白、圆角、层次、指标条」，而非表格本身。

### 3.4 圆角 / 阴影 / 间距

- **圆角**：面板/卡片 `12–14px`；按钮/输入/pill `8px`；小标签 `5–6px`。建立一致的圆角层级（当前几乎无圆角）。
- **阴影**：
  - 面板浮起：`0 1px 3px rgba(26,34,51,.06)`
  - 弹窗浮层：`0 18px 50px -20px rgba(26,34,51,.25)`
  - 聚焦：`0 0 0 2px rgba(79,70,229,.35)`（brand ring）
- **间距**：沿用 UnoCSS 4/8 倍数体系；表格内部保持 `px-2.5 py-[3px]` 紧凑，区块之间用 14–18px。

### 3.5 动效（克制）

- 视图切换：fade（120ms）
- 区块折叠/展开（年度表）：高度过渡（160ms ease-out）
- hover：背景色过渡（100ms）
- 指标条进入：可选的数字 count-up（仅首屏，尊重 `prefers-reduced-motion`）
- 原则：动效服务于状态变化，不滥用微交互

## 4. 整体布局与信息架构（布局 B）

采用「终端化分区」方案，大结构贴近现状，但分区更清晰、补上视觉焦点。

```
┌─────────────────────────────────────────────────────┐
│ 顶部靛蓝强调线                                       │
│ [●●●] FAMILY·FINANCE  方案标签        表格|图表|对比|计算  撤销 重做 ⋯ │ ← 导航条（h-12）
├─────────────────────────────────────────────────────┤
│ 起始 2026·01 │ 结束 2028·12 │ 期限 3年 │ 年化 6.000% │ 初始 ¥500k │ 公积金 开 │ ← 终端式参数行
├─────────────────────────────────────────────────────┤
│ ┌最终累计─┐ ┌期间最低─┐ ┌累计收益─┐ ┌累计支出─┐      │ ← 关键指标条（新增·常驻）
│ │1,027,440│ │ 487,300 │ │  97,440 │ │ 522,000 │      │
│ └─────────┘ └─────────┘ └─────────┘ └─────────┘      │
├─────────────────────────────────────────────────────┤
│ 01  年度汇总                            ▾ 收起       │ ← 编号式区块标题
│ [年度表，可收起]                                     │
├─────────────────────────────────────────────────────┤
│ 02  月度流水                                         │
│ [月度表，主体，获得更多高度]                          │
└─────────────────────────────────────────────────────┘
```

要点：

- **顶部导航条**：精简。左侧 logo（mono 大写 + 三色状态点）+ 方案标签；右侧视图切换（mono 大写 pill，当前项靛蓝底）+ 撤销/重做（图标按钮）+ 更多菜单。顶部一条 2px 靛蓝→透明渐变线作为品牌锚点。
- **终端式参数行**：`起始 / 结束 / 期限 / 年化 / 初始存款 / 公积金` 横排，mono 大写小标签 + 深色值，用细竖线分组。公积金子参数在启用时以「tag」形式跟在后面。
- **关键指标条**：见第 5 节。
- **区块标题**：`01 年度汇总` / `02 月度流水`，mono 编号（靛蓝）+ 中文标题 + 右侧延伸细线。年度表标题带「收起/展开」开关。
- **主区**：表格视图 = 年度表（可收起）+ 月度表（主体）；图表/对比/计算器视图 = 全屏切换（参数行与指标条保留）。
- **年度表可收起**：默认展开，收起后月度表获得更大空间，状态记忆到 localStorage。

> 布局保持「单页、顶部固定导航+参数、下方主内容」的现有骨架，不引入侧边栏（已选布局 B，非 C）。

## 5. 关键指标条（新增·适度增强）

首屏视觉焦点，用现有 `calculate()` 的 `MonthResult[]` 聚合得出，**不新增任何数据字段、不改计算逻辑**，仅加一个纯展示用的 computed。

### 指标定义

| 指标 | 计算（基于 results: MonthResult[]） | 语义色 |
|---|---|---|
| 最终累计 | 末月 `cumSavings` | 中性/brand |
| 期间最低余额 | `Math.min(...cumSavings)` + 出现月份 | warning（若低于初始存款则告警） |
| 累计理财收益 | `sum(investReturn)` | positive（朱砂） |
| 累计总支出 | `sum(monthlyExpense)` | negative（竹青） |

- 公积金启用时，在指标条末尾追加第 5 格「公积金期末余额」（末月 `fundBalance`，琥珀色）；不启用公积金时不显示该格。前 4 格始终保留，不破坏密度。
- 每格：mono 小标签（大写带字距）+ 大号等宽数字 + 灰色小注（如「较初始 +105%」「出现于 2026·04」）。
- 数字智能缩写沿用现有 `format.ts`（万/亿）。

## 6. 各视图详细设计

### 6.1 表格视图（MonthlyTable + AnnualTable）

- **月度表**：列结构完全不变（动态列 → 添加列 → 专项 → 固定列 → 公积金专区 → 对比列）。视觉升级：
  - sticky 表头用 `surface-2` 底 + mono 大写小字表头
  - 极细 `line-soft` 行分隔，斑马纹极淡（`#fafbfd`），hover 行 `brand-50`（靛蓝极淡）
  - **负数从「斜体」改为竹青色**（更直觉，配合已有语义色）
  - 手填单元格 `brand-50` 高亮 + 略加粗
  - 公积金专区用 `border-l-2 brand` 分隔
  - 年末行 `border-b-2` + 加粗
  - 所有交互保留：单元格编辑、上下键移动、右键菜单、列头双击重命名、拖拽重排、hover 公式浮窗、年化分摊 ↻ 标记
- **年度表**：同款视觉语言；项目名列左对齐，数字右对齐；年末存款行 `surface-2` + 加粗强调。
- 区块用 `01 / 02` 编号标题包裹，年度表可收起。

### 6.2 图表视图（FinanceChart）

保留 ECharts，配色与交互升级到新 token：

- 收入柱：朱砂 `#c0504d`；支出柱：竹青 `#6b8e7b`
- 存款主线：靛蓝 `#4f46e5` + 半透明渐变面积
- 公积金副线：琥珀 `#d97706`
- 网格：`line` 极淡；轴线：`neutral-300`
- tooltip：白底卡片化，mono 数字，净结余正负切朱砂/竹青
- 标题区终端化：左侧「财务趋势图 + 当前存款 hero 数字（brand）」，右侧「按月/按年」mono 分段
- 保留按月/按年切换、ResizeObserver 自适应

### 6.3 对比视图（ComparisonView）

- 顶部「方案对比」mono 标题 + 关闭按钮
- 方案勾选行：mono pill，勾选 ≥2 才显示对比表
- 对比表：终端式，指标 × 方案列；每格绝对值 + 与基准差额（朱砂/竹青）
- 表头 `surface-2`，hover `surface-2`，`tabular-nums`

### 6.4 计算器视图（CalculatorView）

- 终端式两栏（房贷等额本息 / 公积金缴存）
- 输入即时计算，合计行 `brand-50` + 加粗
- 与主表格同密度，mono 数字

## 7. 弹窗与浮层统一规范

当前 7 个浮层（FormulaPopover、EventEditor、EventDetailPopover、ContextMenu、MonthPicker、FundFlowEditor、ToolsMenu 下拉）样式高度重复却不统一。统一为：

- 白底 `surface`，圆角 `12px`，阴影 `0 18px 50px -20px rgba(26,34,51,.25)`
- 顶部 mono 小标题（大写带字距）+ 可选关闭 ×
- 内边距统一 14–16px，行距统一
- 关闭交互统一：点击外部 / Esc / mouseleave（按各组件原有触发保留）
- 输入框、按钮走第 8 节组件规范
- 截断/告警统一用 `warning` 琥珀色

各浮层只改外观与内部排版，不改触发逻辑、不改数据流。

## 8. 组件视觉规范

### 按钮（建立主次层级，替代当前"全是描边方块"）

| 类型 | 外观 | 用途 |

| 主按钮 | brand 实心、白字、圆角 8 | 确认/保存/添加 |
| 次按钮 | ghost/描边 `line`、hover `surface-2` | 视图切换、撤销重做 |

| 图标按钮 | ghost 圆角 6，hover `surface-2` | 撤销/重做/关闭/更多 |

视图切换 pill：未选 = `text-3` ghost；选中 = `brand-bg` + `brand-700` 字。

### 输入框

- `surface` 底 + `line` 描边 + 圆角 8
- 聚焦：`brand` 描边 + brand ring
- 数字输入：mono + `tabular-nums` + 右对齐
- 错误态：`danger` 描边 + 红色提示文字

### 标签 / Tag

- `surface-2` 底 + `text-2` 字 + 圆角 5，mono 小字
- 语义 tag：positive/negative/warning 各自浅底深字

### 表格

见 6.1。统一规范：mono 数字、`tabular-nums`、右对齐、极细行线、sticky 表头 `surface-2`。

### 图标系统（替换 emoji）

- 用一套统一的轻量线条 SVG 图标（优先内联 SVG，一致 1.5px 线宽，16px），替换现有 emoji（📥🔄🧮👁）
- 次/装饰位置可保留 mono 符号（↶ ↷ ▾ ＋ × ⋯）
- 不引入重量级图标库（控制打包体积）；实现时可用 `lucide` 风格手绘 SVG 或按需引入 `lucide-vue`（决策留到实现计划）

## 10. 实施策略

采用「先立规范，再并行铺开」，避免并行子代理风格分裂（符合用户偏好并行子代理）。

### 阶段划分

- **阶段 0 · 设计基底**：扩展 `uno.config.ts` token；建立全局 CSS 变量与字体；沉淀一份「组件视觉规范」注释/示例（按钮、输入、表格、弹窗）。**此阶段必须先完成并 review，作为后续所有子代理的统一依据。**
- **阶段 1 · 主框架**：`App.vue` 导航 + 参数行 + 关键指标条 + 布局 B 骨架 + 视图切换样式。
- **阶段 2 · 表格视图**：`MonthlyTable` + `AnnualTable` 新视觉（负数改竹青色等）。
- **阶段 3 · 图表视图**：`FinanceChart` + `financeChart.ts` 配色升级。
- **阶段 4 · 对比 + 计算器**：`ComparisonView` + `CalculatorView`。
- **阶段 5 · 弹窗统一**：7 个浮层统一规范。
- **阶段 6 · 图标 + 收尾**：替换 emoji 为 SVG 图标、滚动条/细节打磨、跨视图一致性巡查。

### 并行策略

- 阶段 0、1 必须串行先完成（确立规范与主框架）
- 阶段 2 / 3 / 4 / 5 在阶段 1 完成后**可并行**，由独立子代理分别承接（每个子代理只动自己负责的组件，无共享文件冲突）
- 阶段 6 收尾串行
- git 操作收归主进程（符合记忆中的用户偏好）

## 11. 验收标准

1. `npm run build` 通过（含 `vue-tsc` 类型检查）
2. `npm run test` 全部通过（所有 `data-testid` 保留，测试不受视觉改造影响）
3. 手动核对：表格 / 图表 / 对比 / 计算器 四个视图视觉语言一致
4. 所有 7 个浮层外观统一
5. 关键指标条数值与表格末行累计一致（交叉验证计算正确）
6. 所有原有功能与交互逐项可用（编辑、拖拽、撤销重做、快捷键、导入导出、快照对比、公积金专区）
7. 首屏有明确视觉焦点（关键指标条），不再「白底无焦点」

## 12. 风险与对策

| 风险 | 对策 |
