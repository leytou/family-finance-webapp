# 月度表右键菜单：清除下方编辑值

## 背景

月度表中有两类可编辑列：

- **动态现金流列**：编辑值存储为列的 `entries[month]`，单元格 `isEdited` 标记。
- **余额列**：编辑值存储为锚点 `anchors`，单元格 `isAnchor` 标记。

用户希望在可编辑列的单元格上右键，弹出自定义菜单，提供"清除该单元格下方所有编辑过的值"的功能，便于批量重置某列从某月起的手工修改。

## 需求确认

| 决策点 | 结论 |
| --- | --- |
| "下方"范围 | **严格下方**：仅清除月份 > 当前行的编辑值，当前单元格保留 |
| 作用列范围 | **仅当前列**：只清除右键所在列下方的编辑值 |
| 余额列是否支持 | **支持**：余额列右键清除下方所有锚点 |
| 二次确认 | **不需要**：点击即清除 |
| 菜单名字 | **清除下方编辑值** |
| 无下方编辑值时 | 菜单项**禁用置灰**（始终显示，不可点） |

## 方案

### 组件结构

新建通用 `ContextMenu.vue` 组件，与表格解耦：

- Props：`x`、`y`（屏幕坐标），`items: MenuItem[]`
- `MenuItem`：`{ label: string; disabled?: boolean; onClick: () => void }`
- 用 `position: fixed` 定位到鼠标坐标
- 点击外部 / Esc 关闭（复用 `useClickOutside`），点击菜单项后执行回调并关闭
- Emits：`close`

业务逻辑（"清除哪些值"）保留在 `MonthlyTable.vue`，菜单组件只负责渲染与定位。

### 交互流程

1. 在可编辑单元格（现金流列单元格、余额列单元格）上监听 `@contextmenu.prevent`。
2. 记录右键的 `columnId`（余额列用特殊标识，如 `'__balance__'`）、`month`、坐标。
3. 计算菜单项：
   - 统计该列下方（月份 > 当前月）编辑过的值的数量 `count`。
   - 菜单项 `清除下方编辑值`，`disabled: count === 0`。
4. 点击菜单项执行清除：
   - 现金流列：遍历 `props.results` 中 `month > 当前月` 且该列 `isEdited` 的行，调用 `store.updateColumnEntry(colId, month, null)`。
   - 余额列：遍历 `month > 当前月` 且 `isAnchor` 的行，调用 `store.removeAnchor(month)`。

### store

现有方法已满足需求，无需新增：

- `updateColumnEntry(colId, month, null)` 删除现金流编辑值
- `removeAnchor(month)` 删除锚点

## 测试

- 现金流列：在某月右键清除，下方编辑值被清除，当前行及上方保留。
- 余额列：在某月右键清除，下方锚点被清除，当前行及上方保留。
- 严格下方边界：当前行的编辑值/锚点不受影响。
- 禁用状态：下方无任何编辑值时，菜单项 `disabled` 为 true。

## 不做的事

- 不做二次确认弹窗。
- 不做"清除整列""清除上方"等其他右键项（保留扩展空间，本次不实现）。
- 不做跨列批量清除。
