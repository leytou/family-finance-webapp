# hover 显示构成公式 设计文档

## 背景

月度表 `MonthlyTable.vue` 已有一套半成品的公式弹窗组件 `FormulaPopover.vue`，但存在以下问题：

- **收入、支出列点击弹出的其实是「本月结余」的公式**：模板里这两个列的 `aria-label` 与事件都复用了 `'monthlyBalance'` 字段。
- **理财公式只写了文字**「上月累计储蓄 × 年利率 / 12」，没有带具体数值。
- 理财/收入/支出/结余四列当前用**单击**触发，与表格里其他「单击=编辑」的交互混杂。
- `FormulaPopover` 的 `FormulaField` 仅含 `'investReturn' | 'monthlyBalance' | 'cumSavings'`，缺少独立的收入/支出公式。

年度表 `AnnualTable.vue` 完全没有公式功能。**注意：年度表已有「专项」行**——`useCalculation.ts` 在有事件的月份向 `columnValues` 注入虚拟列 `{ name: '专项', amount: 该月事件净额 }`，年度表按列名聚合成行，故「专项」会作为一行现金流列出现，但目前无法查看其构成。

## 目标

- **统一交互为 hover**：鼠标悬停显示公式、移开关闭。月表与年表所有公式单元格一致。
- 月度表：**理财、收入、支出、结余、存款**各列 hover 弹出公式。公式按构成项展开并带具体数值。
- 年度表：**年初存款、理财收益、年度结余、年末存款、专项**各行 hover 弹出公式。
- 公式构建逻辑抽成**纯函数**，可单测；弹窗组件退化为**纯展示**。
- 不破坏既有功能。

## 非目标

- 不改月表动态现金流列单元格（单击=编辑值）、月表专项列（单击=打开 `EventEditor`）、月表存款列（单击=编辑锚点）——这些都保留单击语义，公式改用 hover 不与之冲突。
- 不给年度表的普通现金流列各行（如「月薪」行）加公式（它们是原始输入的年度合计，非派生值）；唯独「专项」行加，因其由多个事件派生。
- 不改变 `useCalculation.ts` 的计算口径。

## 口径约定

收入/支出来源（`useCalculation.ts` 已有逻辑）：

- `monthlyIncome` = 所有**启用**列中 `amount > 0` 的求和。
- `monthlyExpense` = 所有**启用**列中 `amount < 0` 的绝对值求和。
- `totalFlow` = 所有启用列 `amount` 求和。
- `monthlyBalance` = `totalFlow + investReturn`。
- `columnValues` 含一个虚拟「专项」列（`id = '__events__'`，`name = '专项'`，`amount` = 该月事件净额），同样参与正负分组。

**月表公式展开规则**（均 hover 触发）：

| 字段 | 公式 |
|------|------|
| `investReturn` | `理财 = 上月存款({prevCum}) × {rate}% ÷ 12 = {investReturn}` |
| `monthlyIncome` | `收入 = {正向列名(金额)} + ... = {monthlyIncome}` |
| `monthlyExpense` | `支出 = {负向列名(金额)} + ... = {monthlyExpense}` |
| `monthlyBalance` | `结余 = 收入({monthlyIncome}) − 支出({monthlyExpense}) + 理财({investReturn}) = {monthlyBalance}` |
| `cumSavings` | 锚点月：`存款 = 锚点值({cumSavings})`；否则 `存款 = 上月存款({prevCum}) + 当月结余({monthlyBalance}) = {cumSavings}` |

**年表公式展开规则**（均 hover 触发）：

| 字段 | 公式 |
|------|------|
| `startSavings` | 首年：`年初存款 = 初始存款({initialDeposit})`；非首年：`年初存款 = 上年年末存款({prevYearEnd})` |
| `investReturn` | `理财收益 = 全年各月理财收益合计 = {investReturn}` |
| `yearBalance` | `年度结余 = {各列名(年度合计) 带正负} + 理财收益({investReturn}) = {yearBalance}` |
| `endSavings` | `年末存款 = 年初存款({startSavings}) + 年度结余({yearBalance}) = {endSavings}` |
| `events` | `专项 = {事件名(金额)} + ... = {全年事件净额}` |

> 年度结余列出全部现金流列带正负号——因年表按列分行、无独立收支汇总列。「专项」行也属于现金流列，会出现在年度结余公式中。

所有金额经 `formatCurrency`（千分位、四舍五入）格式化。

## 架构：纯函数 + 纯展示（方案 A）

### 新增 `src/utils/formula.ts`

导出两个纯函数，输入数据 + 上下文，输出 `{ title: string; lines: string[] }`：

```ts
export type MonthFormulaField =
  | 'investReturn' | 'monthlyIncome' | 'monthlyExpense' | 'monthlyBalance' | 'cumSavings'

export interface MonthFormulaContext {
  annualRate: number      // 年利率，如 0.03；用于理财公式展示
  prevCum: number         // 上月累计储蓄；首月 = initialDeposit
}

export function buildMonthFormula(
  result: MonthResult,
  field: MonthFormulaField,
  ctx: MonthFormulaContext,
): { title: string; lines: string[] }

export type YearFormulaField =
  | 'startSavings' | 'investReturn' | 'yearBalance' | 'endSavings' | 'events'

export interface YearFormulaContext {
  isFirstYear: boolean
  initialDeposit: number
  prevYearEndSavings: number   // 非首年使用；首年传 0
  events: { name: string; amount: number }[]   // 该年全部 MilestoneEvent（仅 events 字段使用）
}

export function buildYearFormula(
  summary: YearSummary,
  field: YearFormulaField,
  ctx: YearFormulaContext,
): { title: string; lines: string[] }
```

`YearSummary` 与 `ColumnSummary` 当前定义在 `AnnualTable.vue` 的 `<script setup>` 内部。为让 `formula.ts` 引用，需把这两个 interface 提取到 `src/types.ts`，与现有 `MonthResult` 并列，便于复用与测试。

`title` 形如 `2026-01 - 收入`（月表，用 `formatMonth`）或 `2026 - 年度结余` / `2026 - 专项`（年表）。`lines` 为公式行，当前每个字段一行；用数组保留多行扩展能力。

### 改造 `src/components/FormulaPopover.vue`

退化为纯展示，props 改为：

```ts
defineProps<{
  title: string
  lines: string[]
  x: number
  y: number
}>()
```

模板渲染 `title` 与 `lines`（每行一个 `<div>`）。保留 `@mouseleave="emit('close')"`。删除 `getFormula`、`formulaLabels`、对 `MonthResult`/`FormulaField` 的依赖。

### 改造 `src/components/MonthlyTable.vue`

- `FormulaField` 类型改为 `MonthFormulaField`（含 `monthlyIncome` / `monthlyExpense` / `cumSavings`），或直接引用 `formula.ts` 的类型。
- `popover` 状态从 `{ result, field, x, y }` 改为 `{ title, lines, x, y }`。
- `showFormula(result, field, event)` 内部：
  1. 计算 `prevCum`：该 result 在 `results` 中的下标 `idx`；`idx === 0 ? initialDeposit : results[idx-1].cumSavings`。
  2. `ctx = { annualRate, prevCum }`，其中 `annualRate`、`initialDeposit` 取自 `store.data.value.systemParams`。
  3. `const { title, lines } = buildMonthFormula(result, field, ctx)`，写入 `popover`。
- **修复**收入列：`field` 由 `'monthlyBalance'` 改为 `'monthlyIncome'`。
- **修复**支出列：`field` 改为 `'monthlyExpense'`。
- 结余列保持 `'monthlyBalance'`，理财列保持 `'investReturn'`。
- **触发方式统一为 hover**：理财/收入/支出/结余四列由当前 `<button @click>` 改为 `<span @mouseenter="showFormula(...)" @mouseleave="popover = null">`（与存款列结构一致，去掉 button）。存款列保持现有 `@mouseenter`。

### 改造 `src/components/AnnualTable.vue`

- 引入 `FormulaPopover` 与 `buildYearFormula`，类型改引 `types.ts` 的 `YearSummary`/`ColumnSummary`。
- 新增 `popover` 状态 `{ title, lines, x, y }` 与 `showYearFormula(summary, field, event)`。
- 聚合 `eventsByYear`（`Map<number, MilestoneEvent[]>`，按 `Math.floor(month/100)` 分组），供专项行公式使用。
- 计算 `ctx`：对点击的 summary 取其在 `yearSummaries` 中的 `index` → `isFirstYear = (index === 0)`，`prevYearEndSavings = index === 0 ? 0 : yearSummaries[index-1].endSavings`，`events = eventsByYear.get(summary.year) ?? []`，`initialDeposit` 取自 `useStore`。
- **四行汇总单元格**（年初存款、理财收益、年度结余、年末存款）与**专项行**单元格：内容包一层 `<span @mouseenter="showYearFormula(summary, <field>, $event)" @mouseleave="popover = null">`。专项行在 `v-for="name in allColumnNames"` 中通过 `name === '专项'` 判定，仅专项行加 `@mouseenter`，其余现金流列行不加。
  - 判定依据：依赖 `useCalculation.ts` 虚拟专项列 `name = '专项'` 这一约定。
- 在 `<template>` 末尾挂载 `<FormulaPopover v-if="popover" ... @close="popover = null" />`。

## 边界情况

- **收入为 0**（无正向列）：`收入 = 0`，`lines` 仍返回单行。
- **支出为 0**（无负向列）：`支出 = 0`。
- **禁用列**：`columnValues` 中 `enabled === false` 的列既不计入统计也不出现在公式。
- **专项净额**（月表）：虚拟专项列按 `amount` 正负分别归入收入/支出展开。
- **专项行（年表）**：仅在某年有事件时出现；hover 公式列出该年全部事件；事件净额正/负原样带号展示。
- **理财公式 prevCum**：锚点月不影响 `investReturn` 计算，`prevCum` 取 `results[idx-1].cumSavings`（首月取 `initialDeposit`）。
- **首月/首年**：分别用 `initialDeposit` 作为 `prevCum` / 年初存款来源。
- **空年表**（无数据）：`yearSummaries` 为空，无单元格可悬停，不影响。

## 测试计划（`tests/formula.test.ts`）

`buildMonthFormula`：

- `monthlyIncome`：多个正向列展开为 `A(x) + B(y) = sum`；负向列不出现。
- `monthlyExpense`：负向列按绝对值展开；正向列不出现。
- `monthlyBalance`：引用收入/支出/理财三项与结果。
- `investReturn`：用 `prevCum` 与 `annualRate` 展示，结果等于 `result.investReturn`（容许四舍五入）。
- `cumSavings`：锚点月与非锚点月两种分支。
- 禁用列不出现在任何公式。
- 收入/支出为 0 时的展示。
- 专项虚拟列参与正负分组。

`buildYearFormula`：

- `yearBalance`：各现金流列带正负号展开 + 理财收益 = 合计。
- `startSavings`：首年显示「初始存款」、非首年显示「上年年末存款」。
- `endSavings`：年初 + 年度结余 = 年末。
- `events`：多个事件展开为 `名(额) + ... = 净额`；与 `summary` 中专项列合计一致。

回归：`npm run test` 全量通过；手动 `npm run dev` 验证月表/年表 hover。
