# 月度表动态列拖拽排序

## 目标

月度表中用户自建的现金流动态列支持按住列头拖拽重新排序，拖拽范围限定在动态列内部（不能拖入或越过固定列：月份、理财、收入、支出、结余、余额，以及可选的对比列）。

## 背景

`MonthlyTable.vue` 的列由两类构成：

- **动态列**：`store.data.value.columns`（`FlowColumn[]`），用户可增删改，列顺序即数组顺序。
- **固定列**：月份、+（添加列）、理财、收入、支出、结余、余额，以及选中快照时的"当时预计""差额"。

列头在 `<thead>`，每行单元格在 `<tbody>`，均通过 `v-for="column in columns"` 渲染。拖拽需联动表头与所有数据行。

## 范围

- 实现动态列在动态列范围内的拖拽重排。
- 拖拽结果持久化（复用现有 localStorage 自动保存）。
- 不改动固定列，不引入拖拽之外的交互变化。

## 架构（3 处改动，职责分离）

1. **`useStore.ts`** — 新增 `moveColumn(fromId, toId, side)`，重排 `columns` 数组，复用现有 `watch(deep)` 自动持久化。
2. **新增 `composables/useColumnDrag.ts`** — 封装拖拽状态机与插入线计算，返回模板可绑定的事件处理函数和响应式状态。提取为独立 composable，因为 `MonthlyTable.vue` 已 670+ 行；符合项目 `useClickOutside` 等 composable 惯例，且可独立单测。
3. **`MonthlyTable.vue`** — 动态列 `<th>` 绑定 `draggable` 与拖拽事件，按状态渲染半透明与插入线；`<tbody>` 整列联动由 Vue 响应式 `columns` 重排自动完成，无需手动操作单元格 DOM。

## 数据流

```
用户按住动态列头拖动
  → useColumnDrag 更新 draggingColumnId / dropTarget（响应式）
  → <th> 渲染：被拖列 opacity-50；目标列左/右边框渲染插入线
拖拽中 dragover：取鼠标 clientX 与目标 th 中点比较 → side = 'before' | 'after'
松手 drop：
  → store.moveColumn(fromId, toId, side)
  → plan.columns 数组重排
  → 现有 watch(deep) 触发 localStorage 自动保存（300ms 防抖）
  → columns computed 重算 → 表头 + 所有数据行整列联动重排
```

## 详细设计

### `store.moveColumn`

```ts
function moveColumn(fromId: string, toId: string, side: 'before' | 'after'): void {
  if (fromId === toId) return              // 拖到自身，无操作
  const plan = getActivePlan()
  const fromIdx = plan.columns.findIndex(c => c.id === fromId)
  if (fromIdx === -1) return
  const [moved] = plan.columns.splice(fromIdx, 1)   // 先移除
  let toIdx = plan.columns.findIndex(c => c.id === toId)  // 移除后重新查找，避免索引错位
  if (toIdx === -1) { plan.columns.push(moved); return }
  if (side === 'after') toIdx += 1
  plan.columns.splice(toIdx, 0, moved)
}
```

并在 `useStore` 返回值中导出 `moveColumn`。

### `useColumnDrag` composable

```ts
// composables/useColumnDrag.ts
import { ref } from 'vue'

export function useColumnDrag(
  reorder: (fromId: string, toId: string, side: 'before' | 'after') => void,
) {
  const draggingColumnId = ref<string | null>(null)
  const dropTarget = ref<{ columnId: string; side: 'before' | 'after' } | null>(null)

  function reset() {
    draggingColumnId.value = null
    dropTarget.value = null
  }

  function onDragStart(columnId: string, e: DragEvent) {
    draggingColumnId.value = columnId
    if (e.dataTransfer) {
      e.dataTransfer.effectAllowed = 'move'
      // setData 让部分浏览器允许 drop 触发
      e.dataTransfer.setData('text/plain', columnId)
    }
  }

  function onDragOver(columnId: string, e: DragEvent) {
    e.preventDefault()                       // 允许 drop
    if (e.dataTransfer) e.dataTransfer.dropEffect = 'move'
    const rect = (e.currentTarget as HTMLElement).getBoundingClientRect()
    const side: 'before' | 'after' =
      e.clientX < rect.left + rect.width / 2 ? 'before' : 'after'
    dropTarget.value = { columnId, side }
  }

  function onDrop(columnId: string, e: DragEvent) {
    e.preventDefault()
    const from = draggingColumnId.value
    const t = dropTarget.value
    if (from && t) reorder(from, t.columnId, t.side)
    reset()
  }

  function onDragEnd() { reset() }

  return { draggingColumnId, dropTarget, onDragStart, onDragOver, onDrop, onDragEnd }
}
```

`MonthlyTable` 中以 `store.moveColumn` 作为 `reorder` 回调初始化：

```ts
const { draggingColumnId, dropTarget, onDragStart, onDragOver, onDrop, onDragEnd } =
  useColumnDrag(store.moveColumn)
```

### `MonthlyTable.vue` 模板与样式

动态列 `<th>`（替换现有动态列 `<th>` 的属性）：

```html
<th
  v-for="column in columns"
  :key="column.id"
  :draggable="renamingColumnId !== column.id"
  :class="[
    'px-1 py-0 text-right tabular-nums font-semibold whitespace-nowrap relative group',
    draggingColumnId === column.id ? 'opacity-50' : '',
    dropTarget?.columnId === column.id
      ? (dropTarget.side === 'before' ? 'drag-line-left' : 'drag-line-right')
      : '',
  ]"
  @dragstart="onDragStart(column.id, $event)"
  @dragover="onDragOver(column.id, $event)"
  @drop="onDrop(column.id, $event)"
  @dragend="onDragEnd"
>
  <!-- 原重命名输入框 / 列名 / 删除按钮结构不变 -->
</th>
```

插入线用 `box-shadow`（不占布局空间，避免列宽跳动）。在 `MonthlyTable` 的 `<style>` 或全局样式中：

```css
.drag-line-left  { box-shadow: -2px 0 0 0 #3b82f6; }
.drag-line-right { box-shadow:  2px 0 0 0 #3b82f6; }
```

## 交互冲突处理

| 现有交互 | 是否冲突 | 处理 |
|---|---|---|
| 双击列头重命名 | 否 | HTML5 DnD 需"按住 + 位移"才触发，纯双击不触发 dragstart，`@dblclick` 正常 |
| 重命名 `<input>` | 是（边缘） | 重命名态下 `:draggable="renamingColumnId !== column.id"` 置 false，避免从 input 内拖动整列 |
| hover × 删除按钮 | 否 | click 不触发 drag；删除按钮保持现有 `@click` 即可 |
| 列名文字选中拖拽 | 否 | th 设 draggable 后拖拽手势由我们接管，浏览器默认文字拖拽被取代 |

## 边界与错误处理

- **0 列**：无动态列 `<th>`，无法拖拽。
- **1 列**：drop 时 `fromId === toId`，`moveColumn` 直接 return，无副作用。
- **拖向固定列**：固定列 `<th>` 无 `draggable`、无 `@dragover.prevent`，drop 不触发，自然无法越界。
- **拖出表格 / 取消**：`dragend` 必然触发 → `reset()` 清理状态。
- **`fromId === toId`**：`moveColumn` 直接 return。
- **不存在的 id**：`findIndex` 返回 -1 → 无操作。
- **数据完整性**：重排只移动数组元素，`FlowColumn` 对象（含 `id`/`name`/`entries`）整体搬运，单元格稀疏数据不受影响。

## 测试方案

### `tests/composables/useStore.spec.ts` — 新增 `moveColumn` 组

- 基本重排 before：`[A, B, C]` → `moveColumn(C, A, 'before')` → `[C, A, B]`
- 基本重排 after：`[A, B, C]` → `moveColumn(A, C, 'after')` → `[B, C, A]`
- `fromId === toId`：顺序不变
- 不存在的 id：顺序不变
- 重排后 `name`/`entries` 数据完整性跟随
- save → reload 顺序保留

### `tests/components/MonthlyTable.spec.ts` — 组件级

- 动态列 `<th>` 渲染 `draggable="true"` 属性
- 固定列（理财/收入/支出/结余/余额）`<th>` 不带 `draggable`（或为 false）
- `wrapper.trigger('dragstart')` + `trigger('dragover')` + `trigger('drop')` 序列后，断言 `store.data.value.columns` 顺序改变
- dragover 后目标列出现插入线 class

> 注：jsdom 下 `DragEvent` 与 `getBoundingClientRect` 需 stub `dataTransfer` 与元素 rect；参考现有 `MonthlyTable.spec.ts` 对事件触动的处理方式。

## 非目标（YAGNI）

- **键盘无障碍重排**：HTML5 DnD 不支持键盘；项目其他交互以鼠标为主，列为 future。
- **触屏拖拽**：本项目为桌面财务工具，HTML5 DnD 的触屏限制可接受。
- **自定义拖拽幽灵图**：默认 `<th>` 幽灵图已足够清晰，无需 `setDragImage`。
- **拖拽动画过渡**：列重排即时生效，不引入过渡动画。
