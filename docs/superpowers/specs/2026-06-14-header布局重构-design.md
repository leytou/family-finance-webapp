# 上方菜单（header）布局重构设计 · 两行分层

日期：2026-06-14

## 1. 背景与问题

当前 header 是单行（`h-12`），右侧一组并排塞了 8 类控件，且混了 4 种性质：

| 性质 | 控件 |
|---|---|
| 参数输入 | 起始月份、年化收益率(%)、初始存款 |
| 历史操作 | 撤销、重做 |
| 视图切换 | 图表、对比 |
| 数据管理 | 「⋯」菜单（导出/导入/重置） |

问题：控件多、性质混杂、无分组，中等宽度窗口即拥挤；左侧方案标签（ScenarioTabs）与右侧控件争抢横向空间。

## 2. 目标与不做

目标：

1. **分层降密度**——把「导航」与「操作」拆到两行，单行控件数减半。
2. **保持可达性**——所有控件仍直接可见（不藏二级面板），参数仍常驻可见。
3. **分组清晰**——同一行的控件性质一致，视觉可辨。

不做（YAGNI）：

- 不做窄屏响应式收纳（两行后单行压力已大减；待真有窄屏需求再加）。
- 不把参数收入「设置」弹层（参数需要常驻可见，本期保持表面）。
- 不改任何计算 / 存储 / 历史逻辑。

## 3. 关键决策（已与用户确认）

| 决策点 | 选择 |
|---|---|
| 整体方向 | **方案 C：两行 header**——第一行导航、第二行操作（优于「分组收纳」与「参数入设置弹层」） |
| 视图切换 | **显式三按钮**（表格 / 图表 / 对比），当前项高亮；移除原「再点一次切回表格」的 toggle |
| 数据菜单 | 触发按钮文案 **「⋯」→「更多」**，下拉内容不变 |
| 撤销 / 重做 | 归第二行右侧，保留 **图标 + 文字**（↶ 撤销 / ↷ 重做） |
| 参数区 | 第二行左侧，前置淡 **「参数」** 小标签标明用途 |
| 两行视觉 | 第一行白底 `h-12`；第二行淡灰底 `bg-neutral-50`、`h-9`，主次分明 |

## 4. 设计方案：两行 header

```
┌──────────────────────────────────────────────────────────────────┐
│ 家庭财务规划 │ [方案A][方案B][+新建][+复制]    [表格][图表][对比] │ 更多 │  ← 第一行 导航（白底）
├──────────────────────────────────────────────────────────────────┤
│ 参数 │ 起始月份[202601]  年化收益率(%)[3.500]  初始存款[100000]   ↶撤销  ↷重做 │  ← 第二行 操作（淡灰底）
└──────────────────────────────────────────────────────────────────┘
```

### 4.1 第一行 · 导航层（白底，`h-12`，`border-b`）

- 左：标题「家庭财务规划」+ 竖线分隔 + `ScenarioTabs`（方案标签、新建、复制）——**不变**。
- 右：视图三按钮 + 竖线分隔 + 「更多」（ToolsMenu）。
  - 视图三按钮：`表格` / `图表` / `对比`，`activeView === 'xxx'` 时高亮（`bg-brand-50 border-brand-200`）。
  - 「更多」：ToolsMenu 触发按钮文案由 `⋯` 改为 `更多`。

### 4.2 第二行 · 操作层（淡灰底 `bg-neutral-50`，`h-9`，`border-b`）

- 左：淡「参数」小标签（`text-[10px] uppercase text-neutral-400` + 右竖线）+ 三个参数输入（组件与绑定**不变**，仅迁移位置）：
  - 起始月份（`MonthPicker`，`v-model="startMonth"`）
  - 年化收益率(%)（`input`，`@input` → `annualRate`）
  - 初始存款（`input`，`v-model.number="initialDeposit"`）
- 右：撤销 / 重做按钮（`data-testid="undo-btn"` / `redo-btn` 不变，`disabled` 绑定 `canUndo`/`canRedo` 不变，`title` 快捷键不变）。

### 4.3 视图切换交互变化（行为变化，重点）

原逻辑（toggle）：

```ts
function showView(view) {
  activeView.value = activeView.value === view ? 'table' : view  // 再点一次回表格
}
```

新逻辑（显式三按钮）：

```ts
function setActiveView(view: ActiveView) {
  activeView.value = view  // 直接设置
}
```

- 新增「表格」按钮，点击 → `activeView = 'table'`。
- `main` 区模板逻辑不变（`comparison` → ComparisonView；`chart` → FinanceChart；`else` → 年/月表）。
- 「表格」按钮取代原「再点图表/对比回表格」的隐式路径，更直观。

## 5. 组件落点清单

| 文件 | 改动 |
|---|---|

  - 「头部包含 ScenarioTabs 和对比按钮」（line 72）——对比按钮仍在，**仍通过**。
  - 撤销/重做用例（line 150+）按 `data-testid` 取按钮，位置迁移**不影响**。
  - **需新增**：断言「表格」按钮存在；点击「表格」按钮能从图表/对比视图回到表格视图（覆盖新的显式三按钮切换）。
- `ToolsMenu` 无单测文件，`⋯→更多` 无测试影响（`App.spec.ts` 也未对其文案断言）。
- 其余 composable / 组件测试不受影响。

## 7. 验收标准

1. header 呈两行：第一行白底含标题 / 方案标签 / 视图三按钮 / 「更多」；第二行淡灰底含「参数」标签 + 三个参数输入 + 撤销/重做。
2. 视图三按钮（表格 / 图表 / 对比）当前项高亮，点击即时切换；「表格」按钮可从图表/对比回到表格。
3. 「更多」按钮展开后仍为导出/导入/重置，功能正常。
4. 撤销/重做仍工作（按钮点击与 `Ctrl+Z` / `Ctrl+Shift+Z` 快捷键），无历史时置灰。
5. 三个参数输入（起始月份 / 年化收益率 / 初始存款）绑定与改前一致，修改后计算结果正确、可撤销。
6. `npm run build` 通过、`npm run test` 全绿（含新增视图切换测试）。
