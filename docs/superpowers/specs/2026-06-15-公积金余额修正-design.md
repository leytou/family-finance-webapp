# 公积金余额修正（对齐存款锚点）设计

日期：2026-06-15
状态：待评审

## 背景与目标

存款支持「修正」：用户在某月点「累计储蓄」格子，直接输入实际储蓄值，后续月份的累计从该实际值接着往下算。用户希望公积金余额也能做同样的修正。

**目标**：让用户能在公积金小窗（`FundFlowEditor`）里输入「实际公积金余额」，覆盖该月预测余额，后续月份以实际值为基准继续累计——交互与存款锚点对齐。

## 现状分析

全链路已排查，结论是**底层已就绪、仅缺 UI 入口**，外加一个历史小 bug：

| 环节 | 存款 | 公积金 |
|------|------|--------|
| 数据模型 | `MonthlyAnchor { month, actualSavings }` | `FundAnchor { month, actualBalance }` ✅ |
| 计算覆盖 | `useCalculation.ts:242-243` `cumSavings = anchor ? actualSavings : ...` | `processFund` 锚点覆盖 `balance = anchor.actualBalance` ✅（`useCalculation.ts:150-152`） |
| 后续月以新值为基准 | ✅ | ✅（同一段循环，下月 `prevBalance` 取本月 `fundBalance`） |
| Store 写入 | `addAnchor` / `removeAnchor` | `addFundAnchor` / `removeFundAnchor` ✅ |
| 数据校验 | `isValidAnchor` | `isValidFundAnchor` ✅ |
| **UI 写入入口** | ✅ `startEditCum`（点格子直接弹输入框） | ❌ **缺失** |
| 已修正高亮 | `isAnchor` 底色 | `isFundAnchor` 底色 ✅ |
| 右键「清除该值」 | ✅ | ❌ **失效**（见下） |

### 发现的 bug：右键「清除该值」对公积金余额列失效

`MonthlyTable.vue` 的 `isCurrentCellEdited`（约 351 行）和 `clearCurrentValue`（约 360 行）只处理了 `BALANCE_COLUMN_ID`（存款），没有处理 `FUND_BALANCE_COLUMN_ID`（公积金余额）。导致：

- `isCurrentCellEdited` 对公积金余额列去 `columns` 里找 `__fund_balance__`，找不到 → 永远返回 `false` → 菜单项「清除该值」永远禁用。
- `clearCurrentValue` 对公积金余额列走 `updateColumnEntry('__fund_balance__', month, null)`，该 id 不是真实现金流列，无效。

（「清除下方公积金锚点」走的是 `clearEditedBelow`，已正确处理 `FUND_BALANCE_COLUMN_ID` → `removeFundAnchor`，不受影响。）

## 设计方案

**做法**：在 `FundFlowEditor` 小窗「= 期末余额」行下方，新增一行「修正为实际余额」输入框。预测期末余额照常显示，下方这行填实际值。两个数字都可见，不混淆。

不采用「把期末余额行直接改成可编辑」的做法——那会让「算出来该多少」和「实际多少」挤在同一处。

## 详细设计

### 1. `FundFlowEditor.vue` —— 新增修正输入框

**UI**：在「= 期末余额」行（当前 120-123 行）下方新增：

```
修正为实际余额  [________]
（留空 = 用上方流水余额）
```

- 该月**未修正**：输入框为空，下方淡色提示「留空 = 用上方流水余额」。
- 该月**已修正**：输入框显示已存的 `actualBalance`。
- 只认数字（沿用 `Number` 解析 + `Number.isFinite` 校验，非法输入忽略，与提取金额一致）。

**新增状态**（镜像现有提取草稿 `rows` / `dirty` 的模式）：

- `draftAnchor: ref<string>` —— 修正框草稿值（字符串）。
- `dirty` 复用：修正框改动也调用 `markDirty()`。
- 初始化：`draftAnchor` 初值 = 入参 `anchorBalance`（`undefined` → 空串，`0` 这种有效零值按字符串保留）。

**提交逻辑扩展 `commit()`**：现有 `commit` 只在 `dirty` 时写提取。扩展为同时处理修正：

- 提取：`replaceMonthWithdrawals(month, items)`（不变）。
- 修正框：
  - `draftAnchor.trim() === ''` → `store.removeFundAnchor(month)`（取消修正）。
  - 非空且 `Number` 合法且 ≠ 原值 → `store.addFundAnchor(month, Math.round(num))`。
  - 值未变化则不写（避免无意义 dirty / 高亮，与存款 `confirmEditCum` 一致）。

提交时机沿用小窗现有约定：点「完成」、点小窗外（`useClickOutside`）、按 Esc（`@keyup.escape="commit"`）。

**交互边界（对齐之前口头说明的微调）**：修正框位于单月浮层内，**不**支持上下方向键逐月跳转（那是表格内联编辑 `startEditCell` / `startEditCum` 才有的能力；浮层里的提取列表同样不支持逐月跳）。回车提交，Esc 提交并关闭。

### 2. `MonthlyTable.vue` —— 修复右键「清除该值」

让 `isCurrentCellEdited` 与 `clearCurrentValue` 也认 `FUND_BALANCE_COLUMN_ID`：

```ts
function isCurrentCellEdited(columnId: string, month: number): boolean {
  if (columnId === BALANCE_COLUMN_ID) {
    return props.results.find(r => r.month === month)?.isAnchor ?? false
  }
  if (columnId === FUND_BALANCE_COLUMN_ID) {
    return props.results.find(r => r.month === month)?.isFundAnchor ?? false
  }
  const column = columns.value.find(c => c.id === columnId)
  return column ? String(month) in column.entries : false
}

function clearCurrentValue(columnId: string, month: number): void {
  if (columnId === BALANCE_COLUMN_ID) {
    store.removeAnchor(month)
  } else if (columnId === FUND_BALANCE_COLUMN_ID) {
    store.removeFundAnchor(month)
  } else {
    store.updateColumnEntry(columnId, month, null)
  }
}
```

### 3. 数据来源：把「该月已修正值」传给小窗

`MonthResult` 只带 `isFundAnchor` 标记，不带 `actualBalance` 数值。给 `FundFlowEditor` 增加一个 prop：

- `MonthlyTable` 新增 helper：`fundAnchorBalance(month): number | undefined`，从 `store.plan.fund.anchors.find(a => a.month === month)?.actualBalance` 取值。
- `FundFlowEditor` 新增 prop `anchorBalance?: number`。
- 模板 `:anchor-balance="fundAnchorBalance(fundFlowEditor.month)"`。

（走 prop 而非在 `FundFlowEditor` 内部查 store，与现有 `withdrawals` 作为 prop 传入保持一致。）

### 4. 不需要改动

- `types.ts`：`FundAnchor` / `FundConfig.anchors` / `MonthResult.isFundAnchor` 均已存在；仅 `FundFlowEditor` props 补一个可选字段（组件内 prop，非全局类型）。
- `useStore.ts`：`addFundAnchor` / `removeFundAnchor` / `isValidFundAnchor` 均已存在。
- `useCalculation.ts`：锚点覆盖与后续月基准已实现。

## 测试计划（Vitest，TDD）

补/确认以下用例：

1. **计算层（已有，确认不回归）**：设 `FundAnchor`，该月 `fundBalance` = `actualBalance`；其后月份从该值继续累计。
2. **`FundFlowEditor`**：
   - 填入实际值 → 完成 → `store` 中 `fund.anchors` 含该月、值为输入值；该月 `result.isFundAnchor === true`。
   - 清空修正框 → 完成 → 该月锚点被移除、`isFundAnchor === false`。
   - 值未变化（原值 = 输入值）→ 完成 → 不写入（无多余锚点 / 无高亮抖动）。
   - 非法输入（字母）→ 忽略，不写入。
3. **`MonthlyTable` 右键**：
   - 公积金余额列已修正时，「清除该值」可用，点击后该月锚点移除。
   - 未修正时「清除该值」禁用。

## 非目标（YAGNI）

- 不改公积金余额格子的左键交互（仍为打开 `FundFlowEditor`）。
- 不在小窗内支持上下键逐月跳转。
- 不引入「批量修正」「按年修正」等新形态。
- 不动存款侧任何逻辑。
