# 初始存款与「余额」改名为「存款」实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 新增「初始存款」作为累计计算的起点本金（从首月起参与投资收益），并将所有用户可见的「余额」文案统一改为「存款」。

**Architecture:** `SystemParams` 新增可选字段 `initialDeposit?: number`；`useCalculation.calculate` 首月 `prevCum` 由 `0` 改为 `initialDeposit ?? 0`，后续月份与年度表/公式弹窗逻辑天然继承，无需改动。`useStore` 在 `createDefault` 设默认 `0`、在 `normalizeWorkspace` 对缺失/非法值补 `0`。UI 在头部新增输入框，并替换月度表/公式弹窗/年度表的「余额」文案。采用**可选字段**方案（相对 spec 的必填定义微调）：现有 `PlanData` 字面量零改动，缺失即 0，语义与 spec「缺失视为可接受、由 normalize 补全」完全一致。

**Tech Stack:** Vue 3 (Composition API + TypeScript) + Vite + UnoCSS + Vitest + @vue/test-utils；持久化 localStorage。

```ts
function normalizeWorkspace(ws: Workspace): Workspace {
  for (const scenario of ws.scenarios) {
    if (!Array.isArray(scenario.plan.snapshots)) {
      scenario.plan.snapshots = []
    }
    // 初始存款缺失或非有限数时补 0（轻量容错，保留存量方案数据）
    if (!isFiniteNumber(scenario.plan.systemParams.initialDeposit)) {
      scenario.plan.systemParams.initialDeposit = 0
    }
  }
  return ws
}
```
