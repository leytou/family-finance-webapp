# 起始月份校验与进位规范化 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让「起始月份」输入框在失焦/回车后始终写入合法 YYYYMM——越界月份（13 月、0 月）按进位规则自动转换，非法输入（位数不足、负数、非数字）回退到旧值。

**Architecture:** 三层分工——`month.ts` 提供纯函数 `normalizeMonth`/`isValidYyyyMm`（复用 `addMonths` 原地进位）；`useStore` 暴露受控动作 `setStartMonth(raw): boolean` 作为唯一写入入口；`App.vue` 把 `v-model.number` 改为 `:value` + `@blur`/`@keydown.enter`，失败时短暂红框、显示值靠单向绑定自动回退。

**Tech Stack:** Vue 3 (Composition API + `<script setup>`) + TypeScript + Vitest + @vue/test-utils (jsdom)。

**对应 spec:** `docs/superpowers/specs/2026-06-12-起始月份校验与进位-design.md`

- [ ] **Step 4: 在 `src/utils/month.ts` 末尾实现两个函数**

```ts
/**
 * 是否为合法的 6 位整数 YYYYMM（年份隐含 1000–9999）。
 * 仅校验位数；月份越界（如 13 月、0 月）交给 normalizeMonth 进位处理。
 */
export function isValidYyyyMm(value: number): boolean {
  return Number.isInteger(value) && value >= 100000 && value <= 999999
}

/**
 * 规范化 YYYYMM：6 位整数则按「年*12+月」原地进位（202613→202701、202600→202512），否则返回 null。
 * 复用 addMonths(offset=0) 实现原地进位，保证与全项目进位规则一致。
 */
export function normalizeMonth(value: number): number | null {
  return isValidYyyyMm(value) ? addMonths(value, 0) : null
}
```

```ts
const startMonthInvalid = ref(false)

function onStartMonthBlur(e: Event) {
  const raw = Number((e.target as HTMLInputElement).value)
  if (!setStartMonth(raw)) {
    startMonthInvalid.value = true
    setTimeout(() => (startMonthInvalid.value = false), 1500)
  }
}
```

```vue
            type="number"
```

改为：

```vue
          <input
            :value="data.systemParams.startMonth"
            @blur="onStartMonthBlur"
```
