# 公积金余额修正 实现计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 让用户能在公积金小窗里输入「实际公积金余额」做修正（对齐存款锚点），并顺带修复右键「清除该值」对公积金余额列失效的问题。

**Architecture:** 底层（数据模型 `FundAnchor`、计算覆盖 `processFund`、Store `addFundAnchor/removeFundAnchor`、已修正高亮 `isFundAnchor`）均已就绪，无需改动。本计划只做两件事：(1) 在 `FundFlowEditor` 小窗「期末余额」下方加修正输入框，复用小窗现有 `commit` 提交；(2) 让 `MonthlyTable` 右键菜单的「清除该值」也认公积金余额列。新增一个 `anchorBalance` prop 把已修正值从 `MonthlyTable` 传给小窗。

**Tech Stack:** Vue 3 (Composition API + `<script setup>`) + TypeScript、UnoCSS、Vitest + @vue/test-utils。

**Spec:** `docs/superpowers/specs/2026-06-15-公积金余额修正-design.md`

改为：

```ts
function commit() {
  if (dirty) {
    const items = rows.value
      .map(r => ({ name: r.name.trim(), amount: Number(r.amount) }))
      .filter(r => r.name !== '' && Number.isFinite(r.amount))
      .map(r => ({ name: r.name, amount: Math.round(r.amount) }))
    store.replaceMonthWithdrawals(props.month, items)
  }
  // 修正为实际余额（锚点）：留空=移除修正；非空且变化=写入；值未变化不写
  const trimmed = draftAnchor.value.trim()
  if (trimmed === '') {
    store.removeFundAnchor(props.month)
  } else {
    const num = Math.round(Number(trimmed))
    if (Number.isFinite(num) && num !== props.anchorBalance) {
      store.addFundAnchor(props.month, num)
    }
  }
  emit('close')
}
```

## 完成后人工核对（非自动测试）

- `npm run dev`，打开有公积金的计划：
  - 点某月公积金余额 → 小窗「期末余额」下方出现「修正为实际余额」输入框。
  - 填一个值 → 完成 → 该月及之后公积金余额按实际值走，月度表余额格高亮。
  - 重新打开该月小窗 → 框里预填上次填的值。
  - 清空框 → 完成 → 回到流水余额，高亮消失。
  - 公积金余额列右键 → 「清除该值」可用（已修正时）/ 禁用（未修正时）。
